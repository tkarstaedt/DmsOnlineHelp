      "Garage Order" Dialog (Vehicle Reservation)

      With the "Garage Order" dialog (also called "vehicle reservation"), planned garage stays for vehicles can be created and edited. A planned garage order blocks the vehicle for dispatching during the specified period.

      Opening the Dialog

      You can open the dialog in the following ways:

      • Create a new order:
        • In the "Garage Orders" view, open the context menu and select Add….
      • Edit an existing order:
        • Double-click an entry in the "Garage Orders" view.
        • Open the context menu of an order in the "Garage Orders" view and selectEdit….

      Description of Dialog Fields

      "Basic Data" Tab

      This tab defines the basic data for the garage order.

      Field Description
      Vehicle Selection of the vehicle to which the order applies.
      Start Planned start of the garage stay (date and time).
      End Planned end of the garage stay (date and time).
      Priority Priority of the order execution (1-10, where 10 is the highest).
      Depot The depot where the work is scheduled to take place.
      Remark Free text field for additional information about the order.
      "Order Data" Tab

      Here, details about the type of order are specified.

      Field Description
      Order Type Selection of the type of garage order.
      Requester Information about the requester.
      Order No. Unique number for the garage order.

      SSB-Specific Mandatory Fields & Requirements

      • Fault Type Category (SSB catalog, e.g., safety-relevant, passenger-affecting)
      • Priority (project-specific scale; influences resource planning)
      • Deadline/Target Date (dependent on fault class and service window)
      • Checklist (SSB template per order type; mandatory for review before release)

      Release & Confirmation Process

      SSB uses a four-eyes principle for defined order types. Changing the status to "Released" requires confirmation from an authorized role (e.g., garage manager). Relevant changes are logged.

      Interfaces & Feedback

      • Import of external messages (fault/diagnosis) to pre-populate fields
      • Feedback (start/end, checklist status) to connected systems
      • KPIs (throughput time, first-fix rate) for reporting
