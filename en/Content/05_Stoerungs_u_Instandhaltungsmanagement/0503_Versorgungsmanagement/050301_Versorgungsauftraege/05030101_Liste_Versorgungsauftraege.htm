      The "Supply Orders" View

      The "Supply Orders" view lists all planned, running, and finished supply orders. These are tasks to be carried out on the vehicle that do not necessarily have to be performed by the (e.g., inspection, cleaning, refueling, sanding).

      For each supply type, the administrator can specify whether the vehicle is blocked for , the standard duration of the execution, and whether the vehicle is automatically reported as finished after the standard duration. A supply order starts when the vehicle is on a track with the corresponding usage type and the current time is within the time range of the order. After completion of the work, the order must be reported as finished so that the vehicle is available again for dispatching.

      Accessing the View

      You can open the list of supply orders in the following ways:

      • Via the "Data Views" toolbar by clicking the Supply Orders button.
      • Via the menu: View > Data Views > Supply Orders .

      View Content

      The list of supply orders typically contains the following columns:

      • Date: The day for which this order is planned.
      • Planned Start: Time at which the supply order is scheduled to begin.
      • Planned End: Time at which the supply order should be completed at the latest.
      • Supply Type: Type of the supply order.
      • Duration: Duration of the supply order in hours, minutes, and seconds.
      • Vehicle: Vehicle number of the vehicle assigned to the order.
      • Vehicle Type: of the vehicle.
      • State: Current state of the order (e.g., Reported, In Progress, Completed, Canceled).
      • Mode: Bus or Rail.
      • Vehicle Depot: Home or current loaned of the vehicle.
      • Location: Current location of the vehicle in the depot.
      • Remark: Additional text for the supply order.
      • Creator: User who created the order.
      • Creation Time: Time at which the order was created.
      • Reported Finished By: User who completed the order.
      • End: Time at which the order was completed.

      Functions in the View

      Various actions can be executed via the (right-click on an order), depending on the order's state:

      • Add: Create a new supply order.
      • Edit: Change an existing order.
      • Report Finished: Report an order as completed.
      • Interrupt: Interrupt a running order (split). The original order is ended at the current time, and a new order is created for a later time.

      A double-click on an entry opens the "Supply Order" dialog for editing.

      The tabular “Supply Orders” lists all planned, running, and completed supply tasks per vehicle (e.g., washing, refueling, interior cleaning). Each order is assigned to exactly one vehicle and one supply type .

      Definition & Purpose

      • Supply Order = Time reservation of a vehicle for an operational supply task (start/end define the reservation period).
      • Goal : Planning, execution, and tracking of non-garage-mandatory work with a dispatching effect.

      Access

      • Toolbar: Data Views → Supply Orders
      • Menu: View → Data Views → Supply Orders
      • Double-click on an entry opens the Supply Order dialog.

      Columns (Standard Configuration)

      • Vehicle (Number), Vehicle Type
      • Start (Operating Day/Time), End (Operating Day/Time)
      • State (Processing Status)
      • Depot (Location of order execution)
      • Supply Type
      • Created by / Created on
      • Reported Finished by / Reported Finished on

      States & Dispatching Effect

      Typical states and their effect:

      • Suggested : Optional time; no dispatching effect.
      • Planned : Bindingly scheduled; vehicle is dispatching-blocked during the reservation period.
      • In Progress : Starts automatically on a suitable supply track or manually; the blocking effect remains until completion (even beyond the planned end).
      • Completed : Supply successfully finished; vehicle is deployable again (automatic feedback by systems, if applicable).
      • Canceled : Not successfully completed; vehicle is also deployable again.

      Operations

      • Add – Create a new supply order (dialog).
      • Edit – Change properties (state-dependent).
      • Execute – From Planned → In Progress .
      • Report Finished – Set to Completed .
      • Cancel – End without successful execution.
      • Split/Interrupt – Divide the order chronologically (creates a follow-up order).

      Integration into Dispatching & Views

      • Dispatching : Considers supply orders with a valid reservation period (/vehicle dispatching is blocked depending on the state/type).
      • Vehicle Dossier : Tab Supply Orders shows all orders related to the vehicle; direct navigation from the list.
      • Depot View : Vehicles with active supplies are visually marked (color coding is project-dependent).

      Supply Track Terminal (VSP) – Optional

      A simplified user interface is available for supply teams. As soon as a vehicle drives onto a supply track, relevant data and actions appear (including reporting a fault as finished, creating a new fault).

      Automatic Generation (Project-Specific)

      Supply orders can also be created automatically (e.g., "Inspection" upon entry into the depot; cleaning dispatching; external systems).

      Overview: "Supply Order" Dialog

      • Basic Data : Vehicle, Vehicle Type, Supply Type, Remark.
      • Scheduling : Start/End, optional All Day , Depot; informal fields Earlier Start/Later End (without dispatching effect).
      • State & Options : Processing status and project-dependent options.
