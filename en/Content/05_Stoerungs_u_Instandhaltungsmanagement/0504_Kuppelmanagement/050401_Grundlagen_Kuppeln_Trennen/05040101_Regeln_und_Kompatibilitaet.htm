      Rules & Compatibility

      This topic describes the rules and compatibility requirements for . It differentiates between hard (binding) criteria and soft (weighted) criteria, explains the checking logic, and shows how conflicts are resolved.

      Fundamentals

      (Type Couple or Decouple) always concern a Terms & Concepts: Coupling Orders. The system checks compatibility before confirmation; the Automatic Vehicle Dispatching uses the same rules for suggestion generation.

      Hard Criteria (Must be fulfilled)

      • Coupling System: mechanically/electrically compatible; coupling height/geometry, if applicable.
      • Technical Approvals: Safety/approvals available; no safety-relevant blocks.
      • Vehicle Class/Profile: Permissible combinations (e.g., length, weight, axle load, steps).
      • Depot/Infrastructure Linkage: Coupling point supports the combination (track/ramp/energy connections).
      • Restriction Flags: Set flags "Do not couple"/"Do not decouple" exclude the opposite action.

      Soft Criteria (Are Weighted)

      • Remaining Range/Energy: SoC/Tank level, predicted range until the next supply.
      • Line Criticality: Priority of important lines/blocks, proximity to departure.
      • Depot Preferences: Home depot, return logic, block extensions.
      • Operational Stability: Minimization of follow-up conflicts, buffer times at the coupling point.
      • Garage/Deadline Status: Next due inspection, soft blocks.

      Checking Logic (Simplified)

      1. Check Hard Criteria  →  not fulfilled = Cancel/Cancellation recommendation with reason.
      2. Score Soft Criteria  →  Evaluation/Recommendation (e.g., "suitable"/"suboptimal").
      3. Collect Conflicts (blocks, supply slots, time/location).
      4. Display Result: Recommendation + Notes; dispatcher can confirm or reject.

      Conflict Scenarios & Measures

      Conflict Detection Recommended Measure
      Incompatible coupling system Hard rule fails Adjust or cancel order; check rule set/configuration
      Restriction flag set (Do not couple/decouple) Vehicle marker exists Respect flag; document cause; seek alternatives
      Supply slot collides Time window overlap Shift coupling time or slot; coordinate with supply department
      Garage block Hard/Soft block status Resolve block or choose another vehicle pair
      Insufficient buffer at coupling point Timetable/calendar check Adjust time window; choose another location; check timetable buffer

      Best Practices

      • One Dialog, Two Modes: Operating logic is identical for coupling and decoupling; the title adjusts (see Dialog: Coupling Order).
      • Before Confirmation check compatibility, blocks, supply slots, and buffers (see Coupling Orders).
      • Consequences of Cancellation: The system sets non-service-preventing markers "Do not couple"/"Do not decouple", which the automatic system bindingly respects.
      • Documentation: Record reasons and decisions in the order; simplifies evaluation and proof.
