      Vehicle Faults – Use Cases from Creation to Completion

      This guide describes the complete workflow for vehicle faults: from creation and editing to creating a garage order and completion.

      Overview of the Fault Workflow

      The lifecycle of a vehicle fault includes the following steps:

      1. Create Fault: Recording the fault with vehicle fault tree, level, and description
      2. Edit Fault: Subsequent adjustment of details (depending on the state)
      3. Create Garage Order: Linking to garage management for resolution
      4. Complete Fault: Report as completed or set to "No Fault Found"

      Not every fault goes through all steps. Minor faults can be marked as "Completed" directly upon creation.

      Create Fault

      Creating a fault is done in the Vehicle Fault dialog and is the first step in the workflow.

      Prerequisites

      • Permission: Fault recording for the respective vehicle
      • Vehicle known: Vehicle identification (number, license plate, or position in view)

      Step-by-Step Guide

      1. Open Vehicle Fault Dialog

      There are several ways to open the dialog:

      Method Description
      From Vehicle List
      1. Vehicles menu → Vehicle List
      2. Select vehicle in the list
      3. Right-click → Create Fault or Create Fault button
      From Vehicle Dossier
      1. Open vehicle dossier (double-click on vehicle)
      2. Faults tab
      3. New Fault button
      From Fault List
      1. Faults menu → Vehicle Faults
      2. New Fault button
      3. Select vehicle in the dialog
      From Dispatching
      1. Select vehicle in dispatching view
      2. Right-click → Report Fault

      Tip: The quickest way is via the vehicle dossier, as all previous faults are directly visible.

      2. Identify Vehicle (if not pre-selected)

      If the vehicle is not yet selected:

      1. Vehicle field: Enter vehicle number, license plate, or name
      2. Select from drop-down list or press F3 for vehicle search
      3. Confirm vehicle

      After selection, the following are loaded automatically:

      • Vehicle type and current dispatching
      • Associated vehicle fault tree (vehicle-type-specific)
      • Existing faults (for information)

      3. Select Fault in Vehicle Fault Tree

      The vehicle fault tree is the central element for categorizing the fault.

      Option A: Navigation through the tree
      1. Select Category (e.g., "Powertrain", "Chassis", "Passenger Area", "Electrics")

      2. Select System within the category (e.g., "Air Conditioning", "Doors", "Engine")

      3. Select Component (e.g., "A/C compressor defective", "Door opener Door 1")

      Option B: Quick entry via fault number
      1. Use Fault No. or Quick Entry field
      2. Enter known fault number (e.g., "425" for "A/C compressor defective")
      3. System automatically navigates to the corresponding entry

      Tip: You can note down frequently used fault numbers or keep them in a quick reference list.

      What happens after selection?

      After selecting a component, the following are automatically adopted (if stored in the vehicle fault tree):

      • Fault level: Suggested value (can be overridden)
      • Fault text: Standard description (can be supplemented)
      • Action note: Notes for garage/driver
      • Capability (negated): If applicable (e.g., "Air Conditioning" for A/C defect)

      4. Set Fault Level

      The fault level (A–E) assesses the vehicle's service level.

      1. Check the suggested fault level
      2. Adjust if necessary:
        • A: No restriction (cosmetic issues)
        • B: Minor restriction (vehicle fully serviceable)
        • C: Limited service level (vehicle only for specific blocks)
        • D: Conditionally serviceable (emergency use only)
        • E: Not serviceable (vehicle blocked)

      Important: The fault level controls availability in dispatching. When in doubt, rate conservatively (higher).

      For more details, see Fault Levels.

      5. Record Description and Remarks

      1. Description field: Check standard text from vehicle fault tree and supplement if necessary
        • Describe symptoms precisely
        • When did the fault occur? (e.g., "since this morning", "during trip on line 42")
        • Has anything already been done?
      2. Remarks field: Additional information (optional)
        • Special circumstances
        • Notes for garage
        • Driver reports

      Best Practice: The more precise the description, the faster the garage can react.

      6. Check Date and Time

      • Date and Time fields are automatically filled with the current timestamp
      • If the fault occurred at an earlier time: Adjust date/time manually

      7. Record Reporter (optional)

      • Reporter field: Name/initials of the person who reported the fault (e.g., driver, garage staff)
      • Often filled automatically with the signed-in user

      8. Immediate Resolution (optional)

      If the fault was already resolved upon creation:

      1. Activate Immediately completed checkbox
      2. Fault is saved directly in the Completed state
      3. No effect on dispatching

      Use case: Fault is discovered and resolved immediately upon creation (e.g., "Loose panel fastened", "Vehicle cleaned").

      9. Save Fault

      1. Click OK or Save button
      2. Fault is saved and receives the state Reported (or Completed, if checkbox is active)
      3. Fault appears in the fault list and vehicle dossier
      4. Fault level becomes active and affects dispatching

      Examples for Different Fault Types

      Example 1: A/C defective

      Vehicle: Bus 4711
      Vehicle fault tree: Passenger Area → Air Conditioning → A/C compressor defective
      Fault level: C (limited service level) – possibly D in summer
      Description: "A/C not cooling. Since yesterday afternoon. Compressor not starting."
      Capability negated: Air Conditioning
      Effect on Dispatching: Vehicle is considered "without A/C", not for blocks requiring A/C

      Example 2: Door defective

      Vehicle: Articulated Bus 8123
      Vehicle fault tree: Passenger Area → Doors → Door opener Door 3 defective
      Fault level: C (limited service level)
      Description: "Door 3 no longer opens. Can be opened manually. Since this morning, line 15."
      Capability negated: Full door function
      Effect on Dispatching: Vehicle only for lines with low passenger frequency

      Example 3: Engine damage (Vehicle not serviceable)

      Vehicle: Bus 5505
      Vehicle fault tree: Powertrain → Engine → Engine damage
      Fault level: E (not serviceable)
      Description: "Engine won't start. Starter motor turns, no ignition. Vehicle is at the depot."
      Effect on Dispatching: Vehicle completely blocked, no dispatching possible

      Common Errors When Creating

      Error Effect Solution
      Vehicle fault tree not navigated to component Imprecise categorization, makes evaluations difficult Always navigate to the lowest level (component)
      Fault level rated too low Vehicle is dispatched although not serviceable When in doubt, rate conservatively (higher)
      Description empty or incomplete Garage cannot reproduce the fault Describe symptoms precisely, state time and circumstances
      Fault accidentally created twice Confusion, duplicate processing Check vehicle dossier before creation to see if fault already exists
      Wrong vehicle selected Fault on the wrong vehicle, real vehicle without fault Check vehicle selection carefully (number, license plate)

      Edit Fault

      Already saved faults can be edited subsequently – however, with restrictions depending on the state.

      Prerequisites

      • Permission: Fault editing for the respective vehicle
      • State: Fault must not be archived

      Step-by-Step Guide

      1. Open Fault

      Method Description
      From Fault List
      1. Faults menu → Vehicle Faults
      2. Select fault in the list
      3. Double-click or Edit button
      From Vehicle Dossier
      1. Open vehicle dossier
      2. Faults tab
      3. Select fault in the list
      4. Double-click or Edit button

      2. Note Differences Between States

      State: Reported

      Editable:

      • Vehicle fault tree selection (Category / System / Component)
      • Fault level
      • Description
      • Remarks
      • Date / Time
      • Reporter

      Locked:

      • Vehicle (cannot be changed)

      Tip: In the "Reported" state, almost all fields are editable. Use this opportunity to correct errors or add details.

      State: In Progress

      Editable:

      • Description
      • Remarks

      Locked:

      • Vehicle
      • Vehicle fault tree selection
      • Fault level
      • Date / Time

      Important: As soon as a garage order has been created (state "In Progress"), central fields are locked. Changes to the fault level or categorization are then only possible via the garage order.

      State: Completed / No Fault Found

      All fields locked. Completed faults can no longer be edited.

      Exception: Administrator permission may allow subsequent corrections (project-specific).

      3. Make Changes

      1. Edit desired fields (depending on state)
      2. Check changes
      3. Click OK or Save button

      Changes take effect immediately:

      • Fault level change → Immediate effect on dispatching
      • Capability change → Recalculation of dispatch vehicle type

      Typical Adjustments

      Adjustment Reason State
      Increase fault level (e.g., B → C) Fault has worsened Reported
      Decrease fault level (e.g., C → B) Temporary fix, vehicle more serviceable again Reported
      Supplement description Additional symptoms occurred or became known Reported, In Progress
      Supplement remarks Garage feedback, intermediate status Reported, In Progress
      Correct vehicle fault tree Wrong category/component selected Reported
      Correct date/time Actual time of fault was earlier Reported

      Permission Requirements

      Action Permission Required
      Edit fault (State: Reported) Fault editing (Standard)
      Edit fault (State: In Progress) Fault editing (Standard) – only description/remarks
      Change fault level in "In Progress" state Extended permissions or via garage order
      Edit completed fault Administrator (project-specific)

      Logging of Changes

      All changes to faults are logged:

      • In the History or Log tab of the dialog
      • Display: Who changed which fields when
      • Traceability for garage and dispatching

      Create Garage Order for Fault

      Linking a fault to a garage order is the central step for resolution. The state automatically changes from Reported to In Progress.

      Prerequisites

      • Permission: Create garage order
      • State: Fault must be in the Reported state

      Three Ways to Create

      Method 1: From the Vehicle Fault Dialog

      1. Open fault (see Vehicle Faults – Use Cases from Creation to Completion)
      2. Click Create Garage Order or Assign button
      3. Garage Order dialog opens:
        • Vehicle is already selected
        • Fault is already linked
        • Description of the fault is adopted
      4. Supplement garage order details (see below)
      5. Save garage order

      Result: Fault automatically changes to In Progress.

      Method 2: From the Fault List

      1. Faults menu → Vehicle Faults
      2. Select one or more faults (multiple selection with Ctrl + click)
      3. Right-click → Create Garage Order or Assign button
      4. Garage Order dialog opens with all selected faults
      5. Save garage order

      Tip: This method is particularly suitable for assigning multiple faults of a vehicle at once.

      Method 3: From the Garage Order Dialog

      1. Garage menu → Garage Orders
      2. New Garage Order button
      3. Select vehicle
      4. In the dialog: Add Faults or Add All Open Faults button
      5. Select faults or automatically adopt all open faults
      6. Save garage order

      Option: Add All Open Faults

      This option is particularly helpful if a vehicle has multiple faults:

      1. In the Garage Order dialog: Add All Open Faults button
      2. System automatically adds all faults in the Reported state
      3. You can remove faults again later (if not all are to be resolved)

      Advantage: Quick assignment without manually searching the fault list.

      Supplement Garage Order Details

      After linking the fault(s), supplement the garage order details:

      Field Description Required
      Order Type Type of maintenance/repair (e.g., "Fault Resolution", "Repair", "Inspection") Yes
      Priority Urgency (e.g., "High", "Normal", "Low") Yes
      Planned Start When should work begin? No
      Planned End When should work be completed? No
      Garage / Processor Who should process the order? No
      Description Is adopted from faults, can be supplemented No

      State Change: Reported → In Progress

      After saving the garage order:

      • All linked faults automatically change to In Progress
      • Fault level remains active (dispatching effect continues)
      • The link to the garage order is displayed in the fault list
      • Vehicle can additionally be blocked by a vehicle reservation (project-specific)

      Automatic Completion when Garage Order is Finished

      As soon as the garage order is finished:

      1. Open garage order dialog
      2. Set status to Completed or Finished
      3. Save garage order

      Automatic consequence:

      • All linked faults automatically change to Completed
      • Fault level becomes inactive (no more effect on dispatching)
      • Negated capabilities are restored
      • Vehicle is fully available again (if no other faults)

      Note: If individual faults were NOT resolved, you can remove them from the order before finishing it. These faults then remain in the "Reported" state.

      Multiple Garage Orders for one Fault?

      Not possible. A fault can only be linked to one garage order.

      Scenario: Fault was assigned, but not resolved → Finish garage order → Fault remains "Reported" → Create new garage order.

      Complete Fault (Completed / No Fault Found)

      A fault is completed either automatically (when garage order is finished) or manually.

      Prerequisites

      • Permission: Complete fault (project-specific: all users or only garage/dispatching)
      • State: Fault must be in Reported or In Progress state

      Two Completion Options

      Option 1: Completed

      Meaning: The fault was successfully resolved. The vehicle is fully serviceable again regarding this fault.

      When to use:

      • Fault was repaired
      • Temporary fix is sufficient
      • Fault was already resolved upon creation (checkbox "Immediately completed")

      Option 2: No Fault Found

      Meaning: The fault was inspected but could not be reproduced. No defect was found.

      When to use:

      • Driver reports noise, garage cannot reproduce it
      • Warning light was active, does not appear during inspection
      • Error message could not be reproduced
      • Fault was already self-resolved before garage took action

      Important: "No Fault Found" does NOT mean the fault is ignored. It is documented and can be traced if it occurs again.

      Manual Ways to Complete

      Method 1: From the Vehicle Fault Dialog

      1. Open fault (see Vehicle Faults – Use Cases from Creation to Completion)
      2. Click Completed or No Fault Found button (depending on option)
      3. Confirm query if necessary
      4. Fault is closed and receives the corresponding state

      Method 2: From the Fault List

      1. Faults menu → Vehicle Faults
      2. Select one or more faults
      3. Right-click → Completed or No Fault Found
      4. Confirm query

      Tip: Multiple selection allows completing several faults at the same time (e.g., all minor damages after a collective garage order).

      Automatic Completion

      Faults are automatically set to Completed when:

      • The linked garage order is finished (see Vehicle Faults – Use Cases from Creation to Completion)
      • The linked vehicle reservation is finished (project-specific)

      Advantage: No manual step necessary. The fault is automatically synchronized with the garage order.

      Effects of Completion

      For "Completed"

      • Fault level becomes inactive → No more effect on dispatching
      • Negated capabilities are restored
      • Dispatch vehicle type is recalculated (based on remaining faults)
      • Fault remains visible in fault list (Filter: "Show completed faults")
      • After configured time: Automatic archiving

      For "No Fault Found"

      • Fault level becomes inactive → No more effect on dispatching
      • Negated capabilities are restored
      • Fault is marked as "No Fault Found"
      • Evaluations can analyze "No Fault Found" faults separately (indicator of sporadic problems)
      • After configured time: Automatic archiving

      Permissions for Completion

      Action Permission Required
      Set fault to "Completed" (State: Reported) Complete fault (project-specific: all or only garage/dispatching)
      Set fault to "Completed" (State: In Progress) Complete fault (often restricted to garage)
      Set fault to "No Fault Found" Set No Fault Found (project-specific: often only garage or garage management)
      Automatic completion (via garage order) Finish garage order

      Project-specific: In some projects, drivers or dispatchers can complete minor faults directly. In others, this is reserved for the garage.

      Examples: When to use which option?

      Scenario Option Reason
      A/C was repaired, works again Completed Fault resolved, vehicle fully serviceable
      Driver reports rattling noise, garage finds nothing No Fault Found Fault could not be reproduced
      Warning light was on, no longer active during inspection No Fault Found Error sporadic, currently not reproducible
      Door was temporarily repaired, works again Completed Temporary fix sufficient for operation
      Fault was already resolved when garage inspected the vehicle No Fault Found or Completed Depending on interpretation: No fault found (not reproducible) or Completed (self-resolved)
      Multiple faults resolved after garage order Completed (automatic) Garage order completion triggers automatic completion

      What happens after completion?

      1. Immediately: Fault level becomes inactive, vehicle dispatchable again (if no other faults)
      2. Short-term: Fault remains visible in the list (for traceability)
      3. After configured time: Automatic archiving
        • Fault is removed from active list
        • Remains available in archive for evaluations and history
        • Typical archiving times: 30–90 days (project-specific)

      Checklist and Best Practices

      Checklist: Create Fault

      • ☐ Vehicle correctly identified (number, license plate)
      • ☐ Vehicle fault tree fully navigated (down to component)
      • ☐ Fault level chosen carefully (conservative if in doubt)
      • ☐ Description complete and precise (symptoms, time, circumstances)
      • ☐ Date/time correct (adjusted to actual time of fault if necessary)
      • ☐ Reporter recorded (who reported the fault?)
      • ☐ Capabilities checked (are correct features negated?)
      • ☐ For immediate resolution: "Immediately completed" checkbox activated
      • ☐ Vehicle dossier checked (are there similar or duplicate faults?)

      Checklist: Create Garage Order

      • ☐ All relevant faults selected (use "Add all open faults" option)
      • ☐ Order type and priority set
      • ☐ Planned times chosen realistically
      • ☐ Processor/garage assigned (if required)
      • ☐ Description adopted from faults and supplemented if necessary
      • ☐ Dispatching informed about vehicle downtime (for longer repairs)

      Checklist: Complete Fault

      • ☐ Fault actually resolved or inspected?
      • ☐ Correct option chosen: "Completed" vs. "No Fault Found"?
      • ☐ For garage order: All faults from the order resolved?
      • ☐ For "No Fault Found": Added remark (why no fault found?)
      • ☐ Dispatching informed that vehicle is available again (for critical faults)

      Best Practices – Overall Workflow

      When Creating

      • Be as precise as possible: The more accurately the fault is described, the faster the garage can react.
      • Use vehicle fault tree completely: Avoid generic categories like "Other".
      • Attach photos: Use the option to attach images or documents to the fault (project-specific).
      • Document immediate resolution: Even small, already resolved faults should be documented (checkbox "Immediately completed").

      When Editing

      • Keep fault level up-to-date: If the situation worsens, increase the fault level.
      • Use remarks: Document intermediate statuses, garage feedback, and findings.
      • Communication: Coordinate with garage and dispatching for critical faults.

      For Garage Order

      • Bundle multiple faults: Use "Add all open faults" to work more efficiently.
      • Set priorities: Mark urgent faults accordingly.
      • Realistic times: Plan planned end realistically to inform dispatching.

      When Completing

      • Check carefully: Is the fault really resolved? Test drive conducted?
      • Document "No Fault Found": Leave notes for future inspections in case of sporadic problems.
      • Inform dispatching: Actively inform dispatching for critical vehicles that they are available again.

      Common Errors and Solutions – Overall Workflow

      Error Effect Solution
      Fault is created but never assigned Fault remains "Reported", vehicle with reduced availability Regular check of open faults, possibly set up automatic reminder
      Garage order finished, but fault remains "In Progress" Vehicle still dispatched with limitations Check if automatic completion is activated (system configuration)
      Fault level not adjusted after worsening Vehicle is dispatched although not serviceable Regular review of faults with long processing status
      "No Fault Found" is used, although fault was resolved Distorted statistics, garage performance not correctly recorded Training: Use "No Fault Found" only for genuinely non-reproducible faults
      Multiple faults created for the same problem Duplicate work, distorted evaluations Check vehicle dossier before creation, edit existing fault if necessary
      Faults set to "Completed" too early Fault recurs, must be created again Perform test drive or thorough inspection before completion
