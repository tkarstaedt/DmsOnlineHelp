      "Vehicle Faults" View

      Displays all reported and processing faults for each vehicle. The fault level assesses the operational service level and is considered by dispatching.

      Overview

      The Vehicle Faults view is the central tool for recording, managing, and tracking vehicle faults (damage/defects). Each row in the view shows the properties of exactly one vehicle-related fault.

      Multiple faults can be managed for a vehicle simultaneously; the operational service level is determined by the highest (most critical) fault level. If there are multiple faults with different fault levels, the highest fault level is displayed in the Fault level column.

      Note: Only current faults are displayed in the view. After a project-configurable time, faults are automatically transferred to the archive database.

      Opening the View

      You can open the Vehicle Faults view in the following ways:

      • Data Views toolbar  →  Vehicle Faults / Faults button
      • View menu  →  Data Views  →  Vehicle Faults

      Columns (Default)

      In the default configuration, the view includes the following columns:

      Column Description
      Vehicle Number of the affected vehicle.
      Vehicle type Vehicle type of the affected vehicle.
      Fault level Degree of the vehicle's operational service level.
      Important: If multiple faults with different fault levels have been recorded for the affected vehicle, the highest fault level is displayed in this column.
      Description / Fault text Short description or further information about the vehicle fault.
      State Current processing status of the vehicle fault (e.g., Created, Reported, In Progress, Completed, No Fault Found).
      See also "Vehicle Faults" View.
      Last changed by User who last processed the vehicle fault.
      Last changed at Time (date and time) of the last processing of the vehicle fault.

      Extended Columns (Optional / Project-Specific)

      Depending on the configuration and project requirements, the following additional columns may be available:

      Column Description
      Fault No. / Message Number Simple text identifier for referencing with external systems (e.g., SAP).
      Mode Bus or Rail.
      Vehicle Depot Home depot or current loaned depot of the vehicle.
      Location Current location of the vehicle in the depot. If the vehicle is outside a depot, this field is empty.
      Remark Additional free text for the fault.
      Creator User who created the fault.
      Creation time / Reported at Time when the fault was created.
      Reported completed by User who completed the fault.
      End time Time when the fault was completed.
      Dispatch vehicle type Due to the fault, a potentially different (calculated) vehicle type for dispatching.

      Note: The column configuration is customizable by the administrator and may vary depending on the project.

      Operation

      Double-Click

      A double-click on a row (fault) opens the Vehicle Fault dialog for editing the selected fault.

      Context Menu

      You can open the context menu by right-clicking on a row (fault). Depending on the state of the fault and your permissions, selected editing calls are available.

      Actions

      The following actions are available via the toolbar and the context menu:

      Action Description Availability
      Add Record a new fault – opens the Vehicle Fault dialog in "New" state. Always
      Edit Open and change the selected fault in the Vehicle Fault dialog. When a fault is selected
      Report finished / Completed Mark fault as completed (state transition to "Completed" / "Finished"). Depending on the fault state
      Cancel / No Fault Found Transition fault to "No Fault Found" state (after unsuccessful inspection). Depending on state and permission
      Assign / Create Vehicle Reservation Convert fault into a garage order or a vehicle reservation.
      Option: "Add all open faults" – all reported faults for the vehicle are assigned to the garage order.      		 For fault in "Reported" state

      Fault States

      Faults can have the following states:

      State Description
      Created The fault is currently being recorded. The recording has not yet been completed with "OK".
      Reported The specific fault has been recorded and saved, but is not yet processed.
      In Progress The fault has been converted into a vehicle reservation or a garage order and is being processed.
      Completed / Finished The resolution of the fault (e.g., repair) was successful. The fault is closed.
      No Fault Found The fault was inspected without being able to reproduce the problem.

      State Transitions

      • Reported → In Progress: By creating a vehicle reservation or a garage order
      • Reported / In Progress → Completed: Manually or automatically (e.g., when the garage order is completed)
      • Reported → No Fault Found: Manually by an authorized user

      Notes on Recording

      Structured Recording via Vehicle Fault Tree

      Use the Vehicle Fault Tree in the Vehicle Fault dialog to record faults in a structured way via Category / System / Component. The vehicle fault tree is vehicle-type-specific and customizable by the administrator.

      Vehicle Fault Dialog

      In the Vehicle Fault dialog, the Basic Data and Details tabs are available. Important fields:

      • Vehicle Fault Tree: Categorization of the fault (Category / System / Component)
      • Fault level:Assessment of the operational service level
      • Fault text: Description of the defect
      • Short text (Driver info): Maximum 40 characters for driver information (project-specific)
      • Action note: Notes for garage / service
      • Remarks: Optional comment

      Completion Actions in the Dialog

      When completing a fault, the following actions are available:

      • Completed: Mark fault as resolved
      • No Fault Found: Close fault without finding
      • Create garage order: Convert fault into a garage order
      • Create vehicle reservation: Convert fault into a vehicle reservation (project-specific)

      Integration into Garage Processes

      Fault → Garage Order

      Via the Assign action, faults are transferred directly into Garage Orders. You have the option "Add all open faults" to assign all reported faults of the vehicle to the garage order.

      When the garage order is completed, the fault automatically changes to the "Completed" state.

      Fault → Vehicle Reservation

      In some projects, faults can be converted into Vehicle Reservations. When the vehicle reservation is completed, the fault is considered resolved.

      Effect on Dispatching

      The fault level controls the availability of the vehicle in automatic and manual block dispatching:

      • Vehicles can be completely or partially blocked for dispatching depending on the fault level.
      • If a vehicle has multiple faults, the highest fault level is used for dispatching.

      Quick Fault Resolution

      Faults that can be resolved immediately should be reported as finished directly after resolution. Use the Report finished or Completed action in the context menu or dialog for this.

      For faults that cannot be reproduced, the No Fault Found state is available.

      Archiving

      Faults are automatically transferred to the archive database after a project-configurable time. Only current faults are displayed in the Vehicle Faults view. The archiving duration is configurable by the administrator.

      Display in Other Views

      Fault information also appears in the following views and dialogs:

      • Vehicle Dossier: Faults for the vehicle, short text next to vehicle state
      • Vehicle List: Fault level display, short text
      • Driver Departure Board: Short text in block information (project-specific)

      Best Practices

      • Quick Faults: Faults that can be resolved immediately should be reported as finished right away.
      • Use Vehicle Fault Tree: Navigate as precisely as possible through the vehicle fault tree for accurate categorization.
      • Choose Fault Level Carefully: The fault level has a direct impact on dispatching.
      • Bundle Multiple Faults: Use the "Add all open faults" option when creating garage orders.
      • Complete Documentation: Record action notes and remarks completely for the garage.
