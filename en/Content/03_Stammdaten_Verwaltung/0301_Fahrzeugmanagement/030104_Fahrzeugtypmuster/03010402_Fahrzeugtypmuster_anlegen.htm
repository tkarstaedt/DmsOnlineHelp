      Create a “Vehicle type pattern”

      Create a pattern of multiple vehicle types and prioritize them for automatic block and parking dispatching.

      Open the “Vehicle type pattern” dialog

      • Toolbar Dialog → Vehicle type pattern
      • Vehicle type patterns view → New in toolbar or context menu → Add

      Fields and meaning

      Field/Element Description
      Designation Unique name of the pattern (required).
      External vehicle type no. Optional reference to external timetabling systems (e.g., DIVA/VDV); may be empty.
      Vehicle type/Priority list Assign permitted vehicle types to the pattern incl. priority 1–10 (controls automatic selection).
      Add/Remove Add types in the list or delete entries you don’t need.
      OK/Apply Save input.

      Step by step

      1. Open the Vehicle type pattern dialog and click New.
      2. Enter a Designation; optionally provide an External vehicle type no.
      3. In the table, choose vehicle types via the selection list and set a priority for each type (1 = low, 10 = high).
      4. Delete entries you don’t need with Remove.
      5. Save with OK.

      Impact on dispatching and parking usage

      • Patterns define a permitted set of vehicle types per block or parking position.
      • Priority controls the order of automatic assignment of suitable types.
      • A vehicle type can be included in multiple patterns.

      Notes

      • Coordinate patterns with external timetabling systems as needed (if an external vehicle type no. is used).
      • Patterns act together with dispatching logic (e.g., capabilities, permitted types per duty lineup position).
