      Sign in to DMS

      Sign in to the PEAK.DMS client — with integrated user management or Active Directory Single Sign-On (SSO) — and optionally choose a user profile or window layout.

      Start the desktop application

      Start the PEAK.DMS client. The User sign-in dialog opens automatically.

      Start the desktop application

      Authentication methods

      • Integrated user authentication: Sign in with DMS Username and Password (built‑in user management).
      • Active Directory (SSO): Sign in as the Windows user without re‑entering name/password (requires IT‑side SSO configuration).

      Sign‑in — step by step

      1. Open the PEAK.DMS client. The User sign-in dialog appears.
      2. For integrated sign‑in: Enter your Username and Password.
      3. Optional: In Your profile, choose a start option:
        • Select: Opens the Select window layout dialog.
        • Last: Starts with the most recently used window layout.
        • Empty: Starts with an empty display area.
        • Selection list: Starts directly with a selected User profile.
      4. Optional: Enable Save password if allowed. Caution: This can pose a security risk.
      5. Click Sign in.

      Advanced sign‑in options

      If required, you can select Authentication method, Project/Database, and Database server. These settings are normally preconfigured and should be changed only by administrators.

      Note: Changes to Project/Server affect the data foundation. Make adjustments only with the appropriate authorization and knowledge.

      Welcome screen & select profile/window layout

      After a successful sign‑in, choose how the application window opens:

      • Restore last window layout — Starts with your last working environment.
      • Start with empty window — Starts with no open views (clears the last used layout).
      • Use a saved profile from the list — Starts with the selected User profile.

      Select window layout or user profile

      Notes on SSO (SSB)

      At SSB, sign‑in typically uses Single Sign‑On. The option Enter user is disabled in the advanced options; the current Windows user is adopted automatically. With Sign in automatically, login can occur at client start without confirmation.
