      View “Block list”

      The Block list shows all blocks for the selected operating day or period; double-click to open the block dossier for details.

      Overview

      The Block list supports quick review of planned/operational blocks and duty assignments, with drill-down to detail dialogs such as the block dossier. Show one operating day or multiple days starting from the selected day; configure the period via the function bar.

      Columns (standard)

      Column Description
      Operating day Calendar date.
      Block Block number; double-click opens the block dossier.
      Start Depot code where the first duty of the block is planned to begin.
      Start (planned) Operating day and time when the vehicle pulls out and the first duty begins (planned).
      Target Depot code where the last duty of the block is planned to end.
      End (planned) Operating day and time when the vehicle pulls in and the last duty ends (planned).
      State Current state of the block (e.g., Planned, In progress, Finished).
      Current vehicle Vehicle currently assigned to the block.

      The columns reflect the standard configuration and may be extended per project.

      States (overview)

      • Planned, In progress, Finished
      • Cancelled (maintained via Cancel/reactivate block in the block dossier)

      Typical actions

      • Open block dossier via double-click or context menu.
      • Create special block via the Special block order button (if available).
      • Cancel block or reactivate cancelled block via context menu or in the block dossier.
      • Maintain vehicle types/pattern in the block dossier (Vehicle types tab; assign pattern).

      Working in the Block list

      Use standard functions of dispatching views: filter/sort/search purposefully and use the context menu to open detail dialogs; set the displayed period in the function bar.

      Drill-down: Block dossier

      In the block dossier, maintain key block data: current vehicle, user-scheduled vehicle, execution note, permitted vehicle types/pattern, block segments, and messages.

      • Set/change current vehicle (Basic data tab).
      • Set user-scheduled vehicle as input to block dispatching.
      • Enter execution note (shown on driver terminal/departure board, if configured).
      • Assign vehicle types/pattern, maintain priorities (input to dispatching).
      • Cancel/reactivate block as permitted by status and time.
