      Troubleshooting Timetable Data & Calendars

      This guide covers common problems related to timetable data, VDV calendar, DMS day types, and DMS calendar, along with their causes, audit trails, and solutions.

      General Notes

      • Work step-by-step: Symptom → Check → Cause → Action.
      • Document manual interventions (date, role, comment) to facilitate later analysis.
      • Compare production and test systems, if available.

      Quick Checklist

      • Import log free of errors? (VDV-452)
      • Base version available and released?
      • Operating day ↔ day type assignment correct?
      • DMS calendar fully updated?
      • Area calendars plausible?

      Common Cases

      Symptom Probable Cause Actions
      Operating days missing in a period DMS calendar not updated or import not performed 1) Dialog "Maintain DMS Calendar" calendar.
      2) Check import status (VDV-452) and re-import if necessary.
      3) Check area calendars.
      Incorrect day type on a date Incorrect assignment in the VDV‑Calendar or incomplete day types 1) Check/complete day type inventory (DMS Day Types).
      2) Correct assignment in the VDV calendar.
      3) In case of planning error: Correct data foundation and redeliver.
      Unexpected operational version active Incorrect or premature release of the base version 1) Check release log.
      2) If necessary, roll back to the last released version (Process: Import to Operational).
      3) Release correct version and document.
      Area calendars differ from DMS calendar Partial update, caching, or filter 1) Refresh view (manually).
      2) Check filter/view.
      3) Verify reference data in the DMS calendar.
      Import reports warnings/errors Structural problems with the delivery (VDV-452) or missing master data 1) Analyze import log.
      2) Add missing master data.
      3) Correct delivery and re-import.

      Audit Trails (Detail)

      1. Import: Check log (error codes, warnings), identify affected elements.
      2. Base Version: Check completeness and validity in the Fahrplandaten anzeigen.
      3. VDV Calendar: Check samples for special days and holidays.
      4. DMS Calendar: Validate preview periods (4 weeks).
      5. Area Calendars: Cross-check Block/Workshop/Supply.

      Prevention & Best Practice

      • Regular calendar updates with scheduling.
      • Change documentation and four-eyes principle for releases.
      • Spot checks before timetable changes and events.
      • Coordination between planning and operations before major changes.
