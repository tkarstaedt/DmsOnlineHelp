      Operating Days

      An operating day is the depot-specific calendar day with an assigned day type. It specifies usage types and permitted vehicle types per stabling position and forms the basis for duty lineup and operational plans.

      Definition and Properties

      The DMS operating day is a depot-specific calendar day assigned a day type in the DMS calendar. It contains the following information:

      • Calendar day-related, depot-specific information on usage types of stabling positions
      • Permitted vehicle types for stabling positions
      • Operational duty lineup plans

      Duration and Operating Day Change

      A DMS operating day always has a duration of 24 hours. The time of an operating day change can be configured per depot (Default: 00:00).

      Best Practice: The BMS should have the same time for the operating day change as DIVA.

      Classification

      • DMS Calendar assigns a day type per depot to a combination of date and depot.
      • Operating day inherits rules (usage types, permitted vehicle types) from the assigned day type.
      • Changes to the operating day apply only to the selected day; the day type remains unchanged.

      DMS Day Types

      The DMS day type is the type of day from the perspective of a DMS dispatcher at a depot. It:

      • Contains depot-specific information on usage types and permitted vehicle types for stabling positions
      • Can be planned independently of specific calendar days
      • Has a unique name

      Default Day Types

      At least one default day type must be defined for each depot and day of the week. The default day types can be found under Tasks / Master Data / Operation.

      Maintain DMS Calendar

      In the DMS calendar, a day type is manually assigned to each combination of operating day (= calendar day) and depot. This allows different depots, if necessary, to use their own day types for the same or different calendar days.

      Accessing Calendar Management

      How to open the BMS/DMS Calendar:

      1. Click the DMS Operating Days button in the Data Views toolbar

      or

      1. Open the DMS Operating Days submenu in the View menu

      Assign Day Types

      1. Open the DMS Calendar dialog.
      2. Select the Depot and assign the Day Type for each date.
      3. Confirm with Apply and close the dialog.

      Note: If an upcoming day (e.g., a Tuesday) is a public holiday, the assignment can be manually adjusted in the DMS Calendar dialog (e.g., to a Sunday day type). Manually adjusted BMS calendar days are not overwritten.

      Creation of Operating Days

      Data Transfer from Day Type

      When the DMS operating day is created, the data from the underlying day type is copied:

      • Usage types
      • Permitted vehicle types
      • Stabling position configurations

      Prerequisite: A day type must already be assigned to the calendar day in the DMS calendar.

      Automatic Creation

      The BMS/DMS calendar must always be defined at least for the preview time of the operational data. A scheduled service is set up as a fallback:

      • A service automatically creates BMS depot calendar days up to and including X days (service parameter) in the future
      • In case of an incomplete calendar, the missing operating days are created automatically
      • The last used day type is assigned as the default
      • Warning messages are displayed in advance if day type assignments are missing

      Automatic Creation & Operational Plans

      • For each created combination of date and depot, an operating day is created with inherited rules.
      • Operational duty lineup plans are generated per operating day from released duty lineup plans of the day types.
      • Blocks are reconciled against the DIVA timetable; pull-out stabling positions and block parameters are transferred.

      Change Behavior

      Changes to the Operating Day

      Usage types, vehicle types, and operational duty lineup plans can be changed for the BMS/DMS operating day. These changes:

      • Apply only to the respective operating day
      • Have no effect on the data of the underlying day type
      • Must be carried out/repeated in the corresponding day type for permanent validity

      Caution: Changes that should apply immediately must be entered for all already created BMS/DMS operating days as well as for the corresponding BMS/DMS day types.

      Example: Change for a Single Operating Day

      November 25, 2008, is assigned the BMS day type "Mon-Thu" at a specific depot.

      • Day Type Rule: Track 26 has usage type Block, only low-floor buses allowed
      • Operating Day Change: On November 25, 2025, low-floor articulated buses are also permitted for Track 26
      • Effect: Only this operating day is affected; neither other daily timetables nor the day type are changed

      Lists and Views

      BMS Day Types

      The BMS Day Types view lists the defined day types per depot with the following information:

      • Designation of the day type
      • Depot
      • Number of linked calendar days

      BMS Operating Days

      The BMS Operating Days view shows the generated operating days per depot and their validity date. Changes to stabling position rules for an operating day only apply to that specific day.

      The data view has the following columns:

      • Depot: Name of the depot for which the operating day applies
      • Calendar Day: Calendar day to which the operating day refers
      • Priorities of Track Usage Types: Priorities of track usage types that apply to this operating day
      • Vehicle Type Priority Lists: Vehicle type priority lists that apply to this operating day
      • Pull-out Phases: Number of pull-out phases scheduled for this operating day

      Stabling Positions for an Operating Day

      Starting from the operating day, the stabling positions with their configurations can be displayed.

      How to view stabling positions for an operating day:

      1. Select an operating day in the BMS Operating Days data view
      2. Call the Stabling Positions context function

      The "Stabling Positions" dialog shows the following fields:

      • Depot: Name of the depot
      • DMS Operating Day: Date of the operating day
      • List of stabling positions with columns:
        • Stabling Position: Name of the stabling position
        • Blocked:Indicates whether the stabling position is blocked
        • Usage Types: Permitted usage types for the stabling position
        • Vehicle Types: Permitted vehicle types for the stabling position

      Operating Day States

      An operating day in the operational database can assume various states. The time periods for state changes are configurable.

      Preview

      Preview data refers to timetable data belonging to future operating days.

      • Automatic timetable generation extends the preview data by one more operating day each day
      • Data can be overwritten by timetable creation if more current planning data is available
      • An existing operating day is always completely replaced; merging existing and newer planning data is not supported
      • Special blocks created by the user are retained or restored

      Caution: Changes already made (e.g., block dispatching) to preview data will be lost during an update and must then be reapplied.

      Fixed

      Data of a fixed operating day:

      • Can only be modified by the user via corresponding dialogs in the PSItraffic client
      • Are no longer changed by the timetable creation process
      • More current VDV planning data is no longer considered during timetable creation
      • If an import of already fixed data is attempted, a warning is displayed to authorized users

      Past

      Data of past operating days:

      • Can no longer be changed
      • Are transferred to the archive database at configurable times
      • Non-archived data is irretrievably deleted from the operational database
      • Archived data is no longer available for operational use and can only be accessed via the Report Manager

      Pull-out Phases

      For display and dispatching purposes, operating days are divided into pull-out phases:

      • A pull-out phase begins at a defined start time.
      • A stabling position can be used for a maximum of one block within the same pull-out phase.
      • The last block in phase t0 pulls out before the first block in phase t1.

      Definition per depot and day type.

      Scenarios (Duty Lineup Planning)

      A scenario refers to a duty lineup plan for a combination of DIVA day type and DMS day type per pull-out phase. Multiple scenarios can exist for one pull-out phase; one is released for operational use.

      Calendar Synchronicity: If day type definitions differ between DIVA and DMS, corresponding combinations must be maintained so that valid duty lineup plans are available for every day of the week.

      Relation to Company Calendar (VDV-452)

      In the context of the VDV-452 interface, the operating day corresponds to the COMPANY CALENDAR:

      • Assignment of the day type to the calendar date of the operating day
      • Only one day type can be assigned to each operating day
      • The table contains an entry for each operating day for which the base version is valid
      • In DMS, this relation corresponds to the OperationDay class

      Best Practices

      • Operating Day Change: The DMS should have the same time for the operating day change as DIVA
      • Default Day Types: At least one default day type must be defined for each depot and day of the week
      • Manual Adjustments: Manually adjusted calendar days (e.g., for holidays) are not overwritten by the automatic service
      • Change Management: Always maintain permanently valid changes in the day type; make one-time changes only on the operating day
