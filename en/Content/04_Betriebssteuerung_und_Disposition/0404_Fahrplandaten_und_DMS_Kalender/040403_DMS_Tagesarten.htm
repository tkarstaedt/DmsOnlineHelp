      DMS Day Types

      The "DMS Day Types" data view shows all DMS day types created in the system, per depot. DMS day types define depot-specific rules (stabling positions, usage types, permitted vehicle types, pull-out phases) and serve as templates for future operating days.

      Overview

      A DMS day type is the type of day from the perspective of a DMS dispatcher at a depot. It contains the depot-specific information on the usage types and permitted vehicle types for the stabling positions, as well as the definition of pull-out phases for the operational sequence.

      DMS day types can be planned independently of specific calendar days and serve as reusable templates for planning operating days.

      Concept and Classification

      Definition

      A DMS day type contains the following depot-specific rules:

      • Stabling positions: Available stabling positions at the depot
      • Usage types: Purpose of each stabling position (e.g., supply, workshop, production, holding area)
      • Permitted vehicle types: Vehicle types that are allowed to be stabled at specific positions
      • Pull-out phases: Time phases during which vehicles leave the depot

      Relationship to Operating Days

      • Operating days are assigned to a DMS day type in the DMS calendar.
      • Upon assignment, operating days automatically adopt the rules of the day type.
      • Changes to a day type only affect future operating days; already created operating days remain unchanged.
      • Multiple DMS day types can be stored for each depot (e.g., Weekday, Saturday, Sunday, Holiday).

      Important: The operating days must be manually linked with the day types in the DMS calendar. Without this link, no operating days can be planned.

      DMS Calendar

      In the DMS calendar, a DMS day type is manually assigned to each combination of operating day (= calendar day) and depot. This allows:

      • Different depots to use their own DMS day types on the same calendar days
      • Different calendar days to have different day types (e.g., Weekday Mon-Fri, Saturday, Sunday)

      The DMS calendar must always be defined at least for the preview time of the operational data in the DMS.

      Opening the Data View

      To open the "DMS Day Types" data view:

      Variant 1: Via Menu

      1. Open the View menu.
      2. Expand the Data Views submenu.
      3. Select the DMS Day Types menu item.
        → The "DMS Day Types" data view is opened.

      Variant 2: Via Toolbar

      1. Activate the Data Views toolbar.
      2. Click the DMS Day Types button.
        → The "DMS Day Types" data view is opened.

      Columns of the Data View

      The "DMS Day Types" data view shows all day types stored in the system per depot in the following structure:

      Column Description
      Designation Name of the day type (e.g., "Weekday", "Saturday", "Holiday", "Sunday").
      The designation should be unique and self-explanatory.
      Depot Assignment of the day type to the depot (applicable depot area).
      Each day type is assigned to exactly one depot.
      Calendar Days Number of calendar days that use this day type in the DMS calendar.
      Note: Day types with a value of "0" are not assigned to any operating day and can be deleted.
      Scenarios Number of linked duty lineup plans (scenarios) per day type.
      Scenarios are defined for duty lineup planning per day type and pull-out phase.

      Operations

      Create DMS Day Type

      You can either create a new DMS day type completely from scratch or duplicate it based on an existing day type.

      Create new day type without a template

      To create a new DMS day type:

      1. Open the DMS Day Types data view.
      2. In the toolbar, click the New button
        or in the context menu (right-click): select Add.
        → The Create DMS Day Type dialog opens.
      3. Designation field: Enter a unique name for the day type (e.g., "Weekday Summer").
      4. Depot field: Select the depot from the selection list.
      5. Click the OK button.
        → The new day type is created.
        → The detail view of the day type opens.
      6. Maintain the rules for the day type:
        • Define usage types per stabling position
        • Specify permitted vehicle types
        • Configure pull-out phases
      7. Save changes.

      Duplicate Day Type

      Duplicating is more efficient than creating from scratch if you want to use a similar day type as a template.

      To duplicate an existing DMS day type:

      1. Open the DMS Day Types data view.
      2. Select the desired source day type in the list.
      3. In the context menu (right-click): select Duplicate
        or in the toolbar: click the Duplicate button.
        → The Duplicate DMS Day Type dialog opens.
      4. Designation field: Enter a new name for the day type.
      5. Depot field: Adjust the depot if necessary (the depot of the source day type is preselected).
      6. Source Day Type field: Is preselected (the previously selected day type).
      7. Click the OK button.
        → The new day type is created with all the rules from the source day type.
        → The detail view of the new day type opens.
      8. Adjust rules as needed and save.

      Efficiency Tip: Use the duplicate function if you want to create a new day type with similar rules to an existing one (e.g., "Weekday Summer" based on "Weekday Winter"). This saves time when configuring stabling positions, usage types, and pull-out phases.

      Delete DMS Day Type

      A DMS day type can only be deleted if it is not assigned to any active operating day.

      To delete a DMS day type:

      1. Open the DMS Day Types data view.
      2. Select the day type to be deleted in the list.
      3. Check the Calendar Days column: The value must be "0".
        → If value > 0: The day type is assigned to operating days and cannot be deleted.
      4. In the context menu (right-click): select Delete
        or in the toolbar: click the Delete button.
        → A confirmation prompt appears.
      5. Confirm the prompt with Yes.
        → The day type is deleted from the system.

      Deletion Restriction: If a day type is assigned to one or more operating days (column "Calendar Days" > 0), an error message is displayed when attempting to delete it:
      "The day type cannot be deleted as it is assigned to active operating days. Please remove the assignment in the DMS calendar first."

      Relationships to Other Objects

      DMS Calendar

      In the DMS calendar, operating days are linked with DMS day types. This link is necessary for the system to apply the depot-specific rules to specific calendar days.

      • The assignment is done manually in the DMS calendar.
      • A DMS operating day always has a duration of 24 hours.
      • The time of change for a DMS operating day can be configured per depot (Default: 00:00).

      Automatic Fallback: In case of an incomplete DMS calendar, the system automatically creates the missing DMS operating days and assigns them the last used DMS day type as a default.

      DMS Operating Days

      Operating days are specific calendar days that have been assigned a day type in the DMS calendar. They automatically adopt the rules of their assigned day type upon creation.

      • Changes to a day type only affect future operating days.
      • Already created operating days remain unaffected by changes to the day type.

      Duty Lineup Planning and Scenarios

      For duty lineup planning, scenarios (duty lineup plans) can be maintained per day type and pull-out phase.

      • The Scenarios column shows the number of linked duty lineup plans per day type.
      • Scenarios can be approved and serve as the planning basis for operational block dispatching.
      • Multiple scenarios (planning variants) can exist per pull-out phase.

      Stabling Positions and Usage Types

      Each day type defines for the assigned depot:

      • Available stabling positions: Number and designation of stabling positions
      • Usage types per stabling position: Purpose (e.g., supply, workshop, production, holding area)
      • Permitted vehicle types per stabling position: Restriction by vehicle type (e.g., only articulated buses, only solo buses)

      Application Examples

      Example 1: Weekday vs. Saturday

      Depot: Depot North

      Day Type "Weekday":

      • 5 pull-out phases (4:30, 5:00, 6:00, 8:00, 12:00)
      • 120 stabling positions available
      • 15 workshop positions
      • Assigned to: Mon-Fri in the DMS calendar

      Day Type "Saturday":

      • 3 pull-out phases (5:00, 7:00, 10:00)
      • 100 stabling positions available
      • 20 workshop positions (more capacity for maintenance)
      • Assigned to: Saturdays in the DMS calendar

      Example 2: Duplicate Day Type for Holiday

      Initial situation: A "Sunday" day type already exists with corresponding rules.

      Task: A special day type with emergency operations needs to be created for Christmas.

      Procedure:

      1. Duplicate the "Sunday" day type
      2. New designation: "Holiday Christmas"
      3. Make changes:
        • Reduction to 2 pull-out phases (emergency operations)
        • Special line clearances
        • Reduced stabling position availability (only 60 positions)
      4. In the DMS calendar: Assign Dec 24, Dec 25, Dec 26 to the new day type

      Important Notes

      • No retroactive changes: Changes to a day type only affect future operating days; already created operating days remain unchanged.
      • Link required: Operating days must be manually linked with day types in the DMS calendar before they can be used.
      • Deletion restriction: Day types can only be deleted if they are not assigned to any active operating day (column "Calendar Days" = 0).
      • Depot-specific: Each day type is assigned to exactly one depot; different depots can have their own day types with the same designation.
      • Multiple use: A day type can be assigned to multiple calendar days (e.g., "Weekday" for Mon-Fri).
      • Duplication saves time: For similar day types, duplicating is more efficient than creating from scratch.
