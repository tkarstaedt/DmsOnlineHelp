      Dispatching Views

      The Dispatching Views in PEAK.DMS bundle all information relevant for daily operations and allow quick drill‑down into detail dialogs (e.g., block dossier, vehicle details, work orders).

      Goal

      You know the purpose and structure of the most important views, use filters effectively, and navigate efficiently to detail information.

      Context

      Dispatching views are built on the principles from Timetable Data and DMS‑Calendar. Day types, operating days, and released timetable versions determine which content is displayed.

      Designations, columns, and menu paths may vary project-specifically. This topic describes the generic product scope.

      Views & Typical Use Cases

      View Purpose Typical Filters Drill‑down
      Vehicle List Quick overview of status, location, assignment. Depot, vehicle group, status (available, locked), day type Vehicle details, work order, supply process
      Block List Check planned/operational blocks and duty assignments. Line, block, time frame, operating day/day type Block dossier dialog, duty details
      Availability Overview Identify available vehicles per time window. Depot, vehicle type, time window, status Vehicle details, dispatching action
      Incidents/Messages Overview of current incidents and affected resources. Priority, area (Block/Garage/Supply), time frame Message details, measures, reference to garage
      Garage Overview Monitor open orders, locks, deadlines. Order status, garage area, vehicle group Work order, history, lock management
      Supply Overview Plan and monitor refueling, charging, cleaning. Supply type, slot/time frame, Depot Process details, slot assignment, resources

      Navigation & Operation

      1. Select the desired view (e.g., Vehicle List).
      2. Set filters and time frame appropriate for the query.
      3. Use column sorting, text search, and grouping (if available).
      4. Open detail dialogs via double-click or context menu.

      Notes & Best Practice

      • Use filters sparingly and save them to quickly call up standard views.
      • Before shift start, check a preview (e.g., 4 hours) for availabilities and incidents.
      • Resolve inconsistencies at the source: timetable data/DMS‑calendar instead of just in the view.
