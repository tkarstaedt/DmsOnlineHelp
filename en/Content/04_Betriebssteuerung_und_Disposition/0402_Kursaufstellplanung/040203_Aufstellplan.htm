      Graphical View "Stabling Lineup Plan"

      The Stabling Lineup Plan is the graphical view of the operational duty lineup plan. It shows tracks with stabling position assignments per pull-out phase and allows vehicle assignment; blocks are not modified here.

      Overview

      The Stabling Lineup Plan view maps the stabling position assignments for a depot and a pull-out phase. A vehicle can be assigned to each block. Block properties (times, structure) are edited in the Duty Lineup Plan or the Operational Duty Lineup Plan; in the Stabling Lineup Plan, blocks are not changed. Gaps indicate deleted blocks.

      Call and Filter

      • Toolbar Data Views  →  Stabling Lineup Plan  or  Menu View  →  Data Views  →  Stabling Lineup Plan.
      • Filter in the toolbar: depot, operating day, pull-out phase.
      • Current View: opens the currently valid pull-out phase for the selected depot.
      • Autom. advance: operating day and pull-out phase automatically advance.

      View Modes

      • Current: shows the vehicles actually connected to the blocks; the background color corresponds to the vehicle state.
      • Planning: shows the vehicles dispatched for the blocks; state colors are not displayed here.
      • Planning (user-scheduled vehicle): shows exclusively stored user-scheduled vehicles; non-existent user-scheduled double tractions appear in the tile color (grey).

      The color coding of the tracks corresponds to the usage type as in the depot view and reflects the current time.

      Content of a Stabling Position Assignment

      • block number  –  red = Tauschkurs (pulling out at another depot).
      • Permissible Vehicle Type Pattern and permissible vehicle types for this duty lineup position (context menu for adjustment; User-defined (Benutzerdef.) = user-defined type set).
      • Pull-out time (Planned) and pull-in time (Planned).
      • Vehicle number of the dispatched or current vehicle; display rules:
        • Color coding as in the depot view (only Current mode).
        • Vehicle already pulled out: italic and struck through.

      Controls

      Assign vehicle

      1. Open the context menu of the stabling position assignment  →  Set Vehicle… (Fahrzeug setzen…).
      2. In the Set Location (Standort setzen) dialog, select a vehicle and confirm with OK. The vehicle is set to the stabling position and assigned to the block.

      Edit permissible vehicle types (duty lineup position)

      1. Open the context menu of the stabling position assignment  →  Edit….
      2. In the duty lineup position dialog, maintain patterns/types and priorities and confirm with OK.

      Effect Context (Wirkungskontext): Call from Duty Lineup Plan affects the BMS‑day type (scenario); call from Operational Duty Lineup Plan/Stabling Lineup Plan only affects the BMS‑operating day.

      Edit usage type/lock of the track

      1. Open the context menu of the track  →  Usage Type (Nutzungsart).
      2. In the Stabling Position Usage Stabling Position (Stellplatznutzung Stellplatz) dialog, adjust usage types, validities, priorities, and optionally locks (multiple uses possible; priority 10 = highest).

      Search and Consistency Check

      • block search: Enter the block number into the search field of the toolbar  →  the block is highlighted in color in the Stabling Lineup Plan.
      • Consistency Check (Konsistenzprüfung): Check the sequence of blocks in the pull-out direction; check result with symbols successful (erfolgreich)/error (Fehler) and details in the Fehlermeldungsbereich.

      Notes

      • Special services are not part of the Duty Lineup Plan. Inclusion in the Operational Duty Lineup Plan occurs automatically by block dispatching or manually from the block list.
      • Locking Period (Sperrfrist): Starting from a parameterized time period before the pull-out time, block/vehicle assignments may no longer be changed.

      SSB – Automatic Stabling Position Assignment and Driver Information

      The system calculates suitable stabling positions (considering supply, maintenance, subsequent service) and transmits target stabling positions and shunting routes to drivers/terminals. In case of conflicts (missing remaining length, lock, impermissible type), alternatives are proposed and deviations are marked on the user interface.
