      "Duty Lineup Plan" Dialog

      Plan the stabling position assignment with blocks per pull-out phase and define permissible vehicle types je duty lineup position. Use scenarios to create variants and release them for operational use.

      Overview

      Der Duty Lineup Plan is the dispatching view for planning the stabling position assignment with blocks in the depot. It serves as the basis for scenarios per pull-out phase and defines the permissible vehicle types per duty lineup position. Planning is independent of the calendar day; changes affect the resulting operational plans for the relevant day type.

      Call and Filter

      • Symbolleiste Dialog  →  Duty Lineup Plan
      • Menü Bearbeiten  →  Duty Lineup Plan
      • Filterzeile: depot, BMS‑Tagesart, pull-out phase as well as, if necessary, scenario wählen.

      Components and Terms

      • pull-out phase: Subdivision of the day type; a maximum of one block is planned per stabling position per pull-out phase.
      • Scenario: Variant status of the duty lineup plan for depot, day type, and pull-out phase; lifecycle Entwurf  →  Prüfung  →  Freigabe. Exactly one scenario is released per triple depot/day type/pull-out phase.
      • duty lineup position: Link between block and stabling position (track) with rules for permissible vehicle types and Priorities.

      Create and Manage Scenarios

      • New: Create scenario (empty or copy) for the selected depot/day type/pull-out phase.
      • Automatically assign: assigns stabling positions to blocks that have not been dispatched.
      • Prüfen: Consistency/plausibility check, marked with symbols erfolgreich/Fehler.
      • Freigeben: release error-free assignment; the released scenario is the template for the operational duty lineup plan.
      • Anordnen: place blocks within the tracks by time/sequence (links/rechts) with a projected minimum distance.
      • Alle zurücksetzen: removes stabling positions of all blocks in the scenario; blocks reappear in the block list.
      • Seite einrichten/Drucken: Print settings and output of the duty lineup plan.

      Block List in the Duty Lineup Plan

      The block list contains the block copies for the selected pull-out phase (Nummer, Linie/Kurs, Vehicle Type Pattern, permissible vehicle types, planmäßige pull-out time). Fahrplanänderungen do not retroactively affect existing scenarios; if necessary, create a new scenario based on the changed Fahrplandaten.

      Track Scheme

      • Only tracks that have the usage type block for the period of the pull-out phase are displayed.
      • Track lengths are shown to scale (projected per depot).
      • Tooltips inform about usage type, permissible vehicle types, and track details; display per Kontextmenü ein/aus.
      • Assignment is permissible if Nutzungsart = block and intersection of permissible vehicle types is given.

      Perform Stabling Position Assignment

      1. Dialog Duty Lineup Plan öffnen and select depot, Tagesart, pull-out phase as well as scenario.
      2. Mark block(s) in the block list.
      3. Per Drag‑and‑Drop into the desired Spur ziehen; the Ablagesymbol shows the placement possibility at the linken Rand.
      4. Move blocks within/between tracks and automatically sort them per Anordnen (by time/sequence, links/rechts).
      5. Achtungssymbol on block tile weist auf Typinkonsistenzen hin (z. B. not approved vehicle types).

      Dispatching and Automation

      The duty lineup plan is closely coupled with the block dispatching. In the freien Aufstellung, blocks can be automatically distributed to available stabling positions and suitable vehicles.

      Validation

      With Validate Duty Lineup Plan, you compare the Fahrplanversion stored in the duty lineup plan with the Fahrplan eines freely selectable operating day. Differences are summarized and displayed line by line.

      Special Services

      Special services are not included in the duty lineup plan. Their stabling position assignment takes place in the operational duty lineup plan automatically by the block dispatching or manually from the block list.

      SSB – Notes

      • Creation is based on temporary Abgleich von DIVA –  and BMS‑Kalender (Tagesarten); Vorschauzeitraum für DIVA is projectable.
      • Per Tripel depot/Tagesart/pull-out phase wird exactly one scenario is released; Freigabe kennzeichnet das scenario for the operative Erzeugung.
