      Validate Duty Lineup Plan

      Check whether the timetable version stored in the duty lineup plan matches the timetable version of an operating day  –  deviations are displayed in a table.

      Overview

      The function Validate Duty Lineup Plan compares the timetable version (base version) stored in the selected scenario (duty lineup plan) with the timetable version of a specified operating day and provides a summary and a list of the deviations.

      Open Dialog

      • Toolbar Dialog  →  Validate Duty Lineup Plan
      • Menu Edit  →  Validate Duty Lineup Plan

      Fields and Meaning

      Field Description
      Scenario Selection of an existing duty lineup plan (scenario) via the “…” button from the list of duty lineup plans.
      Operating day Input/selection of the operating day whose timetable version is to be compared.

      Perform Validation  –  Step by Step

      1. Open the Validate Duty Lineup Plan dialog.
      2. Select a duty lineup plan from the list via the “…” button next to the Scenario field.
      3. Enter or select the Operating day.
      4. Click on Check (Prüfen)  →  the comparison of the timetable versions starts.

      Interpret Result

      • Summary of differences: Overview of the type and number of detected deviations.
      • Deviation list: One row per deviation with details (e.g., missing/omitted blocks, changed times, changed lines/duties, additional blocks).
      • No differences: Duty lineup plan is consistent with the timetable of the operating day.

      Notes

      • The comparison is based on imported VDV‑452 timetable data and their base versions.
      • The timetable generation uses the current base version valid on that day from the planning database for each operating day.
      • In case of deviations: Check the duty lineup plan and re-generate/release it if necessary, or import more current planning data.
