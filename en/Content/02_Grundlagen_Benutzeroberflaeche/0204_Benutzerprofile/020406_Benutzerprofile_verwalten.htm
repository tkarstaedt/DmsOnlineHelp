      Manage user profiles

      In Profile management you edit your own profiles and—if authorized—profiles of other users. Profiles can serve as templates and preserve your window and view layout for recurring tasks.

      Open the “Profile management” dialog

      • Menu File → Manage profiles
      • Or: toolbar File → three‑dot button next to the disk icon
      • Alternatively: menu Window → Manage profiles

      Areas of Profile management

      • Global profiles: Central profiles visible to all users (only administrators may create/edit/delete).
      • Personal profiles: Visible only to the owner and fully controlled by them (administrator can access if needed).
      • Profiles of other users: Optional area requiring additional permissions.

      What does a profile store?

      • Size/position of the application window (including multi‑monitor setups).
      • Which data views/documents are open and their arrangement.
      • Filters, sorting, column settings, and user‑defined map titles in views.

      What does a profile not store?

      • Settings of generic views (configuration/parameters/process data).
      • Menu settings (apply per user regardless of profile).

      Actions

      Load profile

      1. Click Load profile.
      2. Select a profile; currently open views are closed and the selected profile is loaded.

      Save profile

      • Menu Window → Save profile — confirm overwriting the open profile.
      • Or toolbar: Save profile (same effect).

      Create/edit/delete profile

      • Create: Create a new profile and save the current layout.
      • Edit: Adjust layout and save the profile.
      • Delete: Remove profiles you no longer need (global profiles only by administrator).

      Notes & permissions

      • Only with appropriate permissions can you view or edit other users’ profiles.
      • Global profiles are readable by all; editable only by administrators.

      SSB — sign‑in and profile selection

      • After SSO sign‑in, on the welcome screen you can choose the last or a saved user profile; profiles control open views, arrangement, and filters.
      • SSO parameters (session duration/timeout/lockout) follow SSB policy.
