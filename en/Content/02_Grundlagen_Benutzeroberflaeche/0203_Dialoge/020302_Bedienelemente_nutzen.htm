      Use controls

      In dialogs of PEAK.DMS, you work with standardized controls. The following guidance shows how to enter data safely and validate efficiently.

      Note: See related topics for dialog layout, input validation, and efficient workflows.

      Text fields

      • Enter text; field length and allowed characters may be restricted.
      • Required fields are typically marked and validated on save.

      Dropdowns and pickers

      • Open the list and select a value; type‑ahead may be supported.
      • For dates and times, use integrated pickers where available.

      Checkboxes and radio buttons

      • Checkbox: toggle an option on/off.
      • Radio: choose exactly one option from a set.

      Tables and grids

      • Edit inline where permitted; use context menu for row actions.
      • Sorting/filtering/search follows the conventions described in views.

      Save and close

      • Click OK to save input and close the dialog.
      • Click Apply to save input and keep the dialog open.
      • Click Cancel to discard input and close the dialog.
      • Use the Close button in the title bar if no save is required.

      Efficient workflow

      • Work with multiple open dialogs if parallel editing makes sense.
      • Close unneeded dialogs to maintain clarity.
      • Save in steps with Apply.
      • Watch for error messages or confirmation prompts before closing.

      Checklist: Do’s & Don’ts

      Do’s Don’ts
      Save regularly with Apply. Avoid long edits without intermediate saves.
      Use keyboard shortcuts for frequent dialogs. Don’t open too many dialogs at once.
      Read error messages carefully and correct immediately. Don’t ignore warnings or prompts; you could lose data.
      Close unused dialogs to keep the workspace tidy. Don’t leave outdated dialog windows open in the background.

      Reference: Basics of input validation are in Validate inputs and understand feedback.
