      Views — data presentation

      In PEAK.DMS views you sort, filter, and search data, refresh the display, adjust columns, and export results. Depending on the view type, tabular or graphical presentations are available.

      Presentation modes

      • Tabular view: Data in table form — ideal for sorting, filtering, searching, and export.
      • Graphical view: Data as situational picture/diagram/map — focused on visual analysis and context.

      Sort records

      • Click the column header to sort ascending/descending; repeated clicks toggle direction.
      • Multi‑column sorting is available depending on the view.

      Filter records

      Basic filter

      • Show the filter row and enter criteria per column.
      • Multiple columns are logically combined with AND.

      Advanced filter

      • Group conditions and combine AND/OR.
      • Select operators suitable for the data type (text, number, date).

      Search records

      • Ad‑hoc in a single column: focus the column and type the search text.
      • Across all columns: Use the search field (magnifier icon or Ctrl+F); matches are highlighted.
      • Use the arrow at the search field for history; clear filtering with the x in the search field.

      Refresh and status

      • Refresh reloads the data.
      • The status line shows active filters and number of loaded records.

      Adjust columns

      • Change width at the column edge, or double‑click for auto width.
      • Move columns by drag and drop.
      • Show/hide columns via the field/column chooser.

      Save permanent layout adjustments in the user profile (see Customize views).

      Grouping (if available)

      • Toggle grouping by columns to structure large tables.

      Export

      • Export table data, e.g., to Excel.
      • Note: Active filters affect the export (only filtered records are exported).

      Best practices

      • Use filters sparingly and save frequently used settings in your profile.
      • Use time windows/previews appropriate to the question (e.g., availability, incidents).
      • Fix inconsistencies at the data source (e.g., timetable data, DMS calendar), not only in the view.
