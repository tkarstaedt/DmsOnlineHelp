      Display view "Messages"

      The messages view lists all system messages in the depot. Here you get a complete overview of warnings, errors, information, and status messages in PEAK.DMS. Using filter and sort functions, you can narrow the list, acknowledge messages, and view details.

      Purpose and benefits

      • Monitoring of all current system states such as faults, equipment errors, operational events, and notes.
      • Fast filtering by priority, category, object, or time window.
      • Acknowledging messages to document processing.
      • Opening detail dialogs to analyze causes and access linked objects.

      Columns (default)

      The displayed columns can be adapted per project. Typically available:

      Column Description
      No. Unique identification number of the message.
      Timestamp Date and time of occurrence.
      Category e.g., Error, Warning, Information.
      Priority Classification of urgency (high/medium/low).
      Source Causing object (vehicle, stabling position, system module).
      Message text Short description of the event.
      Status Open, acknowledged, or in progress.
      Acknowledged by User who confirmed the message.
      Acknowledged at Time of confirmation.

      Filtering and sorting

      Use the standard functions of tabular views to quickly find messages.

      • Filter: Set conditions (e.g., period, category, or priority) to reduce the display. Multiple filters can be combined.
      • Sort: Click a column header to sort by timestamp, priority, or category.
      • Search: Enter keywords in the search field to search message texts or sources.
      • Group: Group by category or source to present messages in a structured way.

      Acknowledge messages

      Acknowledgement marks messages as noted and/or processed. Procedure:

      1. Select one or more messages in the list.
      2. Click Acknowledge or use the context menu.
      3. Enter a comment if necessary (e.g., cause, responsible person).
      4. Confirm with OK. The status changes to acknowledged and your name appears in the Acknowledged by column.

      Show message details

      • Select the desired message and choose Details (via context menu or double‑click).
      • The detail dialog shows extended information such as parameters, affected objects, history, and possible actions.
      • From here you can navigate directly to linked views (e.g., stabling position, vehicle) or trigger a fault/work order.

      Configuration notes

      • The number and type of displayed messages may be limited by user roles.
      • Color highlighting emphasizes high‑priority messages.
      • Export functions allow saving the list for external analysis.
