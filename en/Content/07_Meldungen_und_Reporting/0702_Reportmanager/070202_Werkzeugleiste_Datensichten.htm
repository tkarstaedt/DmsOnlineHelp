      Toolbar and data views in the Report Manager

      The Report Manager user interface consists of a toolbar, through which you perform actions such as open, save, filter, and export, and of data views in which templates and generated reports are displayed. The following section describes the key elements.

      Toolbar

      The toolbar icons and commands may vary by project. The following functions are common:

      Icon/function Description
      New Creates a new report definition or opens the report editor.
      Open Loads an existing report template for editing.
      Save Saves changes to the current template.
      Refresh Reloads the data and shows changes.
      Filter Opens a dialog for entering filter criteria for the data view.
      Group Toggles the grouping option on or off.
      Export Exports the selected report (e.g., as PDF or CSV).
      Print Sends the report directly to a printer.

      Data views

      The Report Manager typically has two main views:

      • Template view: Shows all available report templates. Here you can create new templates or edit existing ones.
      • Report view: Lists the generated reports (instances). You can filter by date, template, period, or status and open or export the outputs.

      Select a template or report with a double‑click to display details or perform further actions.

      Filtering and grouping

      • Apply filters directly via the toolbar or use the advanced filter form. For example, specify a period, a template, or a category.
      • Use the grouping function to group reports by date, template, or status. This improves the overview with many entries.

      Further notes

      • Your personal settings (columns, sorting, grouping) can be saved as a profile.
      • Administrators can define standard layouts for all users.
      • Changes to report templates only take effect after saving. Enable versioning if needed.
