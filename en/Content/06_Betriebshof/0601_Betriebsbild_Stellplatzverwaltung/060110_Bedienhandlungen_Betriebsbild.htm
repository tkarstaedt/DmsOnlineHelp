      Executing Operations in the Depot View

      This topic bundles common tasks in the area of PEAK.DMS. Each task contains a brief step-by-step procedure and refers to detail topics where necessary.

      Vehicle Search (Ctrl+F)

      Use the to determine the location of a vehicle – in the or on the line.

      Accessing

      • Click on a free area in the → …
      • Edit Menu → …
      • Toolbar → button
      • Keyboard shortcut Ctrl+F

      Search Criteria

      • Vehicle Number
      • Traction
      • Block (the of the current to which a vehicle is signed in are offered)

      Result

      • Vehicle is in the Depot: The corresponding is opened and the vehicle is briefly highlighted (flashing).
      • Vehicle is in Service: The opens with the last reported location from the RBL.

      Block Parking Position

      1. Open the (see “Depot View”) or the list of parking positions (see “Parking Positions” View).
      2. Select → Edit… (opens the , see Dialog "Edit Parking Position").
      3. On the Parking Position Data tab, activate the Blocked option, set Reason for Block, and optionally Valid From/To.
      4. Select OK to save.

      Unblock Parking Position

      1. Open the “Edit Parking Position” Dialog (see Dialog "Edit Parking Position").
      2. Deactivate the Blocked option and clear the time period/reason, if applicable.
      3. Select OK.

      Add Usage Type

      1. Select → Edit… (see Dialog "Edit Parking Position").
      2. Open the Usage Types tab → Add….
      3. Select (e.g., Block, Garage, Supply, Reserve, Storage Space), optionally set color/time period.
      4. Confirm with OK.

      Remove Usage Type

      1. In the Usage Types tab, select the row to be removed.
      2. Select Remove → Confirm → OK.

      Allow Vehicle Type

      1. Open (see Dialog "Edit Parking Position").
      2. Vehicle Types tab → Add….
      3. Select (and pattern, if applicable), optionally add a Note.
      4. Select OK.

      Remove Vehicle Type

      1. In the Vehicle Types tab, select the row.
      2. Select Remove → OK.

      Compose Driver Information

      Driver information is used for targeted communication (e.g., on , changed paths). Availability is project-specific.

      1. Open the list (see “Driving Orders” View) or select a vehicle/ in the (see “Depot View”).
      2. → Compose Driver Information… (designation may vary).
      3. Enter subject and message, define target group/recipient, set validity/confirmation requirement, if applicable.
      4. Select Send or OK.

      Create Repositioning Move

      Move a vehicle plan-able between parking positions (Source → Target). Detailed fields and options can be found in the dedicated topic.

      1. Start from or list () or directly via orders (see “Repositioning Move” Dialog).
      2. Define vehicle, Source, Target, Start, and optional End.
      3. Set Priority and Reason, select Execution Type → OK.

      Notes

      • Permissions: Some actions require write permissions or specific roles.
      • Project-specific fields/names are possible; follow your configuration.
      • E-Bus: Consider charging window, SOC, and charging zones when selecting the target.
