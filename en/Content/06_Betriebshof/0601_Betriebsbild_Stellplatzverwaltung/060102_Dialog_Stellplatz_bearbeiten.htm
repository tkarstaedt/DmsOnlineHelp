      Dialog "Edit Parking Position"

      In the “Edit Parking Position” , you change properties of a parking position, such as designation, block status, usage types, or permitted . The dialog is usually opened from the “” or the “Parking Positions” .

      Opening the Dialog

      1. From the : Double-click the parking position.
        Alternatively: Right-click → Edit Parking Position…
      2. From the “Parking Positions” : Select the parking position in the table and click Edit… (or double-click the record).

      SSB-Specific Fields & Specifications

      • Priority per Usage Type/Vehicle Type: SSB uses a 1–10 scheme (10 = highest priority) for competing assignments.
      • Block Status Parking Position/Track: Blocked parking positions are not assigned by ; marked via lock symbol.
      • Type Permissibility: / lists control automatic and manual assignments.
      • Special Areas: “Non-plausible location” (Quarantine) is not assigned by .

      : Changes in the Day Type context affect future operational plans; in the operating day context, only day-related.

      Structure and Tabs

      The dialog is structured into tabs. The following tabs are available in the standard configuration:

      • Parking Position Data – Basic data, visibility, block status.
      • Usage Types – Operational usage and color coding.
      • Vehicle Types – Allowed /patterns.
      • Messages – Triggering events for notes/messages (optional).

      “Parking Position Data” Tab

      Maintain the general properties of the parking position. Mandatory fields are marked.

      Field Description
      Designation Display name in the and in lists.
      Type Single parking position, track segment, or storage space.
      Area/Track Assignment within the (e.g., A/03).
      Visible Controls whether the parking position is displayed in the .
      Blocked Excludes the parking position from .
      Reason for Block Short text for (e.g., construction work).
      Valid From/To Time period for block or special usage.
      Note Internal comment (may appear as a tooltip, if applicable).

      “Usage Types” Tab

      Assign operational usage types to the parking position. Order may determine priority in .

      Column Description
      Usage Type E.g., Block, Garage, Supply, Reserve, Storage Space.
      Active Switches the usage on/off.
      Color Display color in the (project-specific).
      Valid From/To Optional time period.

      Actions: Add…, Edit…, Remove, Move Up/Down.

      “Vehicle Types” Tab

      Define which (and patterns) are allowed to occupy the parking position.

      Column Description
      Vehicle Type Allowed type (e.g., Solo Bus, Articulated Bus, Tram Type X).
      Pattern Optional pattern.
      Note Short comment (e.g., length restriction).

      Actions: Add…, Remove.

      “Messages” Tab (optional)

      Configure event-based messages for the parking position.

      • Message Type: Information, Warning, Fault.
      • Trigger: Assignment changed, rule violated, block ends, etc.
      • Recipient/Target: Display in DMS, driver information, reporting (project-specific).

      Typical Actions

      The following work steps are particularly frequent in the :

      1. Block Parking Position: Activate Blocked, enter Reason for Block and Valid From/To, save.
      2. Add Usage Type: Open the Usage Types tab → Add… → select type → optionally set time period → save.
      3. Allow Vehicle Type: Vehicle Types tab → Add… → select type (and pattern, if applicable) → save.

      E-Bus Specific Notes (optional)

      Additional fields may be available for charging and supply areas.

      • Charging Point/Charging Station: Assignment to charging infrastructure (e.g., connector/position).
      • Operational Usage “Supply”: Clearly mark positions reserved for charging/refueling procedures.

      Save and Close

      • OK: Save changes and close the .
      • Cancel: Discard changes.

      SSB: Rule Sets for Parking Position Assignment

      • Priority per Usage Type/Vehicle Type (project-specific scale; high number = high priority)
      • Type Permissibility (Classes/Lengths; influence on automatic assignment & suggestion)
      • Block Status (temporary/permanent; not assignable by )
      • Reservations (e.g., charging window/ window)

      Shunting Window & Path Clearance (SSB)

      Shunting windows and path clearances are coordinated with rules. In case of conflicts (assignment, oncoming traffic, blocks), the system suggests alternatives and marks deviations.
