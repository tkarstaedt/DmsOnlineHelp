      “Driving Orders” View

      The tabular “Driving Orders” displays all planned or running orders in the in PEAK.DMS, e.g., , moves, and moves. It is used for prioritization, processing, and control.

      Opening the View

      1. Open the navigation tree and select Depot > .
      2. Alternatively: Search for “” and open the from the search results.

      Columns (Standard)

      The available columns are project-specific. Typical columns in the standard configuration:

      Column Brief Description
      Order Number Unique identifier of the order.
      Type e.g., , , .
      Vehicle Vehicle number/type.
      Source Start /area.
      Target Target /area.
      Start Planned start (date/time).
      End Planned end/deadline (optional).
      Priority Urgency level.
      Status e.g., planned, in progress, completed, canceled.
      Creator User/context.
      Note Short comment.

      Filtering and Searching

      Use the standard functions of tabular to quickly find orders.

      • Filter: e.g., Status = planned, Type = , Vehicle = 1234.
      • Search: Free text, substrings, column search.
      • Date/Time: Limit the time window for start/end.

      Sorting and Grouping

      Adapt the display to the work context.

      • Sort: Click column header (e.g., by Start or Priority).
      • Group: Group by Status, Type, or Vehicle.
      • Show/Hide Columns: Tailor the to the information needs.

      Context Menu (Selection)

      Actions are available via right-click on a data record.

      • Open/Edit… – Open the order and adjust fields.
      • Cancel – Cancel the order and provide a reason.
      • Duplicate – Copy the order as a template, adjust fields.
      • Show in Depot View – Center source/target in the (see “Depot View”).
      • New Repositioning Move… – Open the for creation (see Repositioning Move (Dialog)).
      • History – Show change log (if available).

      Typical Work Steps

      1. Prioritize Orders: Sort by Priority → process critical orders first.
      2. Check Conflicts: Identify and resolve overlaps in vehicles/time windows.
      3. Create Repositioning Move: → New Repositioning Move… (see Repositioning Move (Dialog)).

      E-Bus Specific Notes

      Additional constraints must be observed for orders related to charging.

      • Charging Window/SOC: Consider time dependencies related to charging points.
      • Target Selection: Prefer charge-capable ; observe reservations.

      Export and Print

      • Export: list for evaluation/reporting.
      • Print: Output the current as a work list.
