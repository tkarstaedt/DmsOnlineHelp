DMS-Tagesarten

Die Datensicht "DMS-Tagesarten" zeigt alle im System angelegten DMS-Tagesarten je Betriebshof. DMS-Tagesarten definieren betriebshofspezifische Regeln (Stellplätze, Nutzungsarten, zulässige Fahrzeugtypen, Ausrückphasen) und dienen als Vorlage für künftige Betriebstage.

Übersicht

Eine DMS-Tagesart ist die Art eines Tages aus Sicht eines DMS-Disponenten eines Betriebshofes. Sie enthält die betriebshofspezifischen Informationen zu den Nutzungsarten und zulässigen Fahrzeugtypen der Abstellplätze sowie die Definition von Ausrückphasen für den Betriebsablauf.

DMS-Tagesarten können unabhängig von konkreten Kalendertagen geplant werden und dienen als wiederverwendbare Vorlagen für die Planung von Betriebstagen.

Konzept und Einordnung

Definition

Eine DMS-Tagesart enthält folgende betriebshofspezifische Regeln:

  • Stellplätze: Verfügbare Abstellplätze auf dem Betriebshof
  • Nutzungsarten: Zweckbestimmung je Stellplatz (z. B. Versorgung, Werkstatt, Produktion, Stauraum)
  • Zulässige Fahrzeugtypen: Fahrzeugtypen, die auf bestimmten Stellplätzen abgestellt werden dürfen
  • Ausrückphasen: Zeitliche Phasen, in denen Fahrzeuge den Betriebshof verlassen

Beziehung zu Betriebstagen

  • Betriebstage werden im DMS-Kalender einer DMS-Tagesart zugeordnet.
  • Bei der Zuordnung übernehmen Betriebstage automatisch die Regeln der Tagesart.
  • Änderungen an einer Tagesart wirken nur auf künftige Betriebstage; bereits erzeugte Betriebstage bleiben unverändert.
  • Für jeden Betriebshof können mehrere DMS-Tagesarten hinterlegt werden (z. B. Werktag, Samstag, Sonntag, Feiertag).

Wichtig: Die Betriebstage müssen im DMS-Kalender manuell mit den Tagesarten verknüpft werden. Ohne diese Verknüpfung können keine Betriebstage geplant werden.

DMS-Kalender

Im DMS-Kalender wird manuell jeder Kombination aus Betriebstag (= Kalendertag) und Betriebshof eine DMS-Tagesart zugewiesen. Damit können:

  • Verschiedene Betriebshöfe zu gleichen Kalendertagen eigene DMS-Tagesarten verwenden
  • Unterschiedliche Kalendertage verschiedene Tagesarten haben (z. B. Werktag Mo-Fr, Samstag, Sonntag)

Der DMS-Kalender muss immer mindestens für die Vorschauzeit der operativen Daten im DMS definiert sein.

Datensicht aufrufen

So öffnen Sie die Datensicht "DMS-Tagesarten":

Variante 1: Über Menü

  1. Menü Ansicht öffnen.
  2. Untermenü Datensichten aufklappen.
  3. Menüpunkt DMS-Tagesarten wählen.
    → Die Datensicht "DMS-Tagesarten" wird geöffnet.

Variante 2: Über Symbolleiste

  1. Symbolleiste Datensichten aktivieren.
  2. Schaltfläche DMS Tagesarten klicken.
    → Die Datensicht "DMS-Tagesarten" wird geöffnet.

Spalten der Datensicht

Die Datensicht "DMS-Tagesarten" zeigt alle im System hinterlegten Tagesarten je Betriebshof in folgender Struktur:

Spalte Beschreibung
Bezeichnung Name der Tagesart (z. B. "Werktag", "Samstag", "Feiertag", "Sonntag").
Die Bezeichnung sollte eindeutig und selbsterklärend sein.
Betriebshof Zuordnung der Tagesart zum Betriebshof (geltender Depotbereich).
Jede Tagesart ist genau einem Betriebshof zugeordnet.
Kalendertage Anzahl der Kalendertage, die diese Tagesart im DMS-Kalender verwenden.
Hinweis: Tagesarten mit Wert "0" sind keinem Betriebstag zugeordnet und können gelöscht werden.
Szenarien Anzahl der verknüpften Kursaufstellpläne (Szenarien) je Tagesart.
Szenarien werden für die Kursaufstellplanung pro Tagesart und Ausrückphase definiert.

Bedienhandlungen

DMS-Tagesart anlegen

Sie können eine neue DMS-Tagesart entweder vollständig neu anlegen oder auf Basis einer vorhandenen Tagesart duplizieren.

Neue Tagesart ohne Vorlage anlegen

So legen Sie eine neue DMS-Tagesart an:

  1. Datensicht DMS-Tagesarten öffnen.
  2. In der Symbolleiste auf Schaltfläche Neu klicken
    oder im Kontextmenü (Rechtsklick): Hinzufügen wählen.
    → Dialog DMS-Tagesart erzeugen öffnet sich.
  3. Feld Bezeichnung: Eindeutigen Namen für die Tagesart eingeben (z. B. "Werktag Sommer").
  4. Feld Betriebshof: Betriebshof aus der Auswahlliste wählen.
  5. Schaltfläche OK klicken.
    → Neue Tagesart wird angelegt.
    → Detailansicht der Tagesart wird geöffnet.
  6. Regeln der Tagesart pflegen:
    • Nutzungsarten je Stellplatz definieren
    • Zulässige Fahrzeugtypen festlegen
    • Ausrückphasen konfigurieren
  7. Änderungen speichern.

Tagesart duplizieren

Das Duplizieren ist effizienter als eine Neuanlage, wenn Sie eine ähnliche Tagesart als Vorlage nutzen möchten.

So duplizieren Sie eine vorhandene DMS-Tagesart:

  1. Datensicht DMS-Tagesarten öffnen.
  2. Gewünschte Quell-Tagesart in der Liste markieren.
  3. Im Kontextmenü (Rechtsklick): Duplizieren wählen
    oder in der Symbolleiste: Schaltfläche Duplizieren klicken.
    → Dialog DMS-Tagesart duplizieren öffnet sich.
  4. Feld Bezeichnung: Neuen Namen für die Tagesart eingeben.
  5. Feld Betriebshof: Bei Bedarf Betriebshof anpassen (vorausgewählt ist der Betriebshof der Quell-Tagesart).
  6. Feld Quell-Tagesart: Ist vorausgewählt (die zuvor markierte Tagesart).
  7. Schaltfläche OK klicken.
    → Neue Tagesart wird mit allen Regeln der Quell-Tagesart angelegt.
    → Detailansicht der neuen Tagesart wird geöffnet.
  8. Bei Bedarf Regeln anpassen und speichern.

Effizienz-Tipp: Nutzen Sie die Duplizier-Funktion, wenn Sie eine neue Tagesart mit ähnlichen Regeln wie eine bestehende Tagesart erstellen möchten (z. B. "Werktag Sommer" basierend auf "Werktag Winter"). Dies spart Zeit bei der Konfiguration von Stellplätzen, Nutzungsarten und Ausrückphasen.

DMS-Tagesart löschen

Eine DMS-Tagesart kann nur gelöscht werden, wenn sie keinem aktiven Betriebstag zugeordnet ist.

So löschen Sie eine DMS-Tagesart:

  1. Datensicht DMS-Tagesarten öffnen.
  2. Zu löschende Tagesart in der Liste markieren.
  3. Spalte Kalendertage prüfen: Der Wert muss "0" sein.
    → Falls Wert > 0: Tagesart ist Betriebstagen zugeordnet und kann nicht gelöscht werden.
  4. Im Kontextmenü (Rechtsklick): Löschen wählen
    oder in der Symbolleiste: Schaltfläche Löschen klicken.
    → Sicherheitsabfrage erscheint.
  5. Sicherheitsabfrage mit Ja bestätigen.
    → Tagesart wird aus dem System gelöscht.

Lösch-Restriktion: Ist eine Tagesart einem oder mehreren Betriebstagen zugeordnet (Spalte "Kalendertage" > 0), wird beim Versuch, die Tagesart zu löschen, eine Fehlermeldung angezeigt:
"Die Tagesart kann nicht gelöscht werden, da sie aktiven Betriebstagen zugeordnet ist. Bitte entfernen Sie zunächst die Zuordnung im DMS-Kalender."

Beziehungen zu anderen Objekten

DMS-Kalender

Im DMS-Kalender werden Betriebstage mit DMS-Tagesarten verknüpft. Diese Verknüpfung ist erforderlich, damit das System die betriebshofspezifischen Regeln auf konkrete Kalendertage anwenden kann.

  • Die Zuordnung erfolgt manuell im DMS-Kalender.
  • Ein DMS-Betriebstag hat immer eine Dauer von 24 Stunden.
  • Der Zeitpunkt des Wechsels eines DMS-Betriebstags kann pro Betriebshof konfiguriert werden (Standard: 00:00 Uhr).

Automatische Rückfallebene: Das System erstellt im Falle eines unvollständigen DMS-Kalenders die fehlenden DMS-Betriebstage automatisch und weist ihnen als Standard die zuletzt verwendete DMS-Tagesart zu.

DMS-Betriebstage

Betriebstage sind konkrete Kalendertage, denen im DMS-Kalender eine Tagesart zugeordnet wurde. Sie übernehmen bei der Anlage automatisch die Regeln ihrer zugeordneten Tagesart.

  • Änderungen an einer Tagesart wirken nur auf zukünftige Betriebstage.
  • Bereits erzeugte Betriebstage bleiben von Änderungen an der Tagesart unberührt.

Kursaufstellplanung und Szenarien

Für die Kursaufstellplanung können pro Tagesart und Ausrückphase Szenarien (Kursaufstellpläne) gepflegt werden.

  • Die Spalte Szenarien zeigt die Anzahl der verknüpften Kursaufstellpläne je Tagesart.
  • Szenarien können freigegeben werden und dienen als Planungsgrundlage für die operative Umlaufdisposition.
  • Pro Ausrückphase können mehrere Szenarien (Planungsvarianten) existieren.

Stellplätze und Nutzungsarten

Jede Tagesart definiert für den zugeordneten Betriebshof:

  • Verfügbare Stellplätze: Anzahl und Bezeichnung der Abstellplätze
  • Nutzungsarten pro Stellplatz: Zweckbestimmung (z. B. Versorgung, Werkstatt, Produktion, Stauraum)
  • Zulässige Fahrzeugtypen je Stellplatz: Einschränkung nach Fahrzeugart (z. B. nur Gelenkbusse, nur Solo-Busse)

Anwendungsbeispiele

Beispiel 1: Werktag vs. Samstag

Betriebshof: Depot Nord

Tagesart "Werktag":

  • 5 Ausrückphasen (4:30, 5:00, 6:00, 8:00, 12:00 Uhr)
  • 120 Stellplätze verfügbar
  • 15 Werkstatt-Stellplätze
  • Zugeordnet zu: Mo-Fr im DMS-Kalender

Tagesart "Samstag":

  • 3 Ausrückphasen (5:00, 7:00, 10:00 Uhr)
  • 100 Stellplätze verfügbar
  • 20 Werkstatt-Stellplätze (mehr Kapazität für Wartungsarbeiten)
  • Zugeordnet zu: Samstage im DMS-Kalender

Beispiel 2: Tagesart für Feiertag duplizieren

Ausgangssituation: Es gibt bereits eine Tagesart "Sonntag" mit entsprechenden Regeln.

Aufgabe: Für Weihnachten soll eine spezielle Tagesart mit Notbetrieb erstellt werden.

Vorgehen:

  1. Tagesart "Sonntag" duplizieren
  2. Neue Bezeichnung: "Feiertag Weihnachten"
  3. Änderungen vornehmen:
    • Reduzierung auf 2 Ausrückphasen (Notbetrieb)
    • Spezielle Linienfreigaben
    • Reduzierte Stellplatzverfügbarkeit (nur 60 Plätze)
  4. Im DMS-Kalender: 24.12., 25.12., 26.12. der neuen Tagesart zuordnen

Wichtige Hinweise

  • Keine rückwirkende Änderung: Änderungen an einer Tagesart wirken nur auf zukünftige Betriebstage; bereits erzeugte Betriebstage bleiben unverändert.
  • Verknüpfung erforderlich: Betriebstage müssen im DMS-Kalender manuell mit Tagesarten verknüpft werden, bevor sie genutzt werden können.
  • Lösch-Restriktion: Tagesarten können nur gelöscht werden, wenn sie keinem aktiven Betriebstag zugeordnet sind (Spalte "Kalendertage" = 0).
  • Betriebshofspezifisch: Jede Tagesart ist genau einem Betriebshof zugeordnet; verschiedene Betriebshöfe können eigene Tagesarten mit gleicher Bezeichnung haben.
  • Mehrfachverwendung: Eine Tagesart kann mehreren Kalendertagen zugeordnet werden (z. B. "Werktag" für Mo-Fr).
  • Duplizierung spart Zeit: Bei ähnlichen Tagesarten ist das Duplizieren effizienter als die Neuanlage.
