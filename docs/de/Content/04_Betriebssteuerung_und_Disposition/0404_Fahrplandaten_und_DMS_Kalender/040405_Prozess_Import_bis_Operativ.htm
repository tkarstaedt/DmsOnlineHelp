Prozess: Import bis Operativ

Dieser Prozess beschreibt den Weg der Fahrplandaten von der Planung (z. B. via VDV-452) über die Basisversion und die Freigabe bis in die operative Datenbasis von PEAK.DMS.

Ziel

Sie können Fahrplandaten importieren, inhaltlich prüfen, freigeben und sicher in den operativen Betrieb übernehmen – inklusive Dokumentation und Rücksprungmöglichkeit.

Rollen & Verantwortlichkeiten

Rolle Aufgabe im Prozess
Planung Erstellt/lieferte den Export (z. B. VDV-452); definiert Tagesarten.
Disponent Prüft RBL-Kalender/Basisversionen, bewertet Auswirkungen auf Disposition.
Fachadmin DMS Startet Import, führt Qualitätschecks durch, gibt Version frei oder verwirft sie.

Ablauf (Kurzüberblick)

  1. Import der Lieferungen aus der Planung (z. B. VDV-452).
  2. Bildung einer Basisversion und inhaltliche Prüfung.
  3. Freigabe der gewünschten Version.
  4. Übernahme nach Operativ und Prüfung der Bereichsansichten.
  5. Protokollierung und ggf. Rollback.

Schritte im Detail

1. Import

  1. Öffnen Sie die Importfunktion und wählen Sie das Lieferpaket (z. B. VDV-452-Daten).
  2. Starten Sie den Import und prüfen Sie das Ergebnisprotokoll auf Fehler/Hinweise.
  3. Es entsteht eine Basisversion der Fahrplandaten (siehe Fahrplandaten anzeigen).

2. Inhaltliche Prüfung

  • Kontrollieren Sie im RBL-Kalender und den Basisversionen die Gültigkeitszeiträume und Vollständigkeit.
  • Prüfen Sie die Zuordnung Betriebstag ↔ Tagesart im VDV‑Kalender sowie die DMS-Tagesarten.
  • Bewerten Sie Auswirkungen auf Dialog „Kursaufstellplan“ und Fahrzeugdisposition  –  Einführung.

3. Freigabe

  1. Wählen Sie die geprüfte Basisversion aus.
  2. Erteilen Sie die Freigabe für den operativen Einsatz.
  3. Dokumentieren Sie Datum/Uhrzeit, verantwortliche Rolle und Kommentar.

4. Übernahme nach Operativ

  1. Stoßen Sie die Übernahme in die operative Datenbasis an.
  2. Prüfen Sie die Bereichsansichten (z. B. Kalender: Umläufe, Werkstatt, Versorgung) auf Plausibilität.
  3. Validieren Sie stichprobenhaft Kursaufstellplan und Dispositionssichten.

5. Protokollierung & Rollback

  • Jede Freigabe/Übernahme wird protokolliert (Version, Zeitpunkt, Rolle, Kommentar).
  • Bei Bedarf können Sie auf die zuletzt freigegebene Version zurückspringen (Rollback).
  • Fassen Sie Gründe und Folgen der Rücknahme in einem Kommentar zusammen.

Qualitätssicherung (Checkliste)

  • Vollständige Betriebstage im DMS-Kalender? → siehe Dialog „DMS-Kalender pflegen“
  • Korrekte Tagesarten-Zuordnung? → siehe VDV‑Kalender
  • Bereichskalender plausibel? → Umlauf/Werkstatt/Versorgung geprüft
  • Kein offener Fehler im Importprotokoll?

Häufige Probleme & Lösungen

  • Fehlende Betriebstage: Kalender fortschreiben und erneut prüfen (siehe Dialog „DMS-Kalender pflegen“).
  • Falsche Tagesart: Zuordnung im VDV-Kalender korrigieren.
  • Unerwartete operative Version aktiv: Freigaben prüfen; ggf. Rollback durchführen.
