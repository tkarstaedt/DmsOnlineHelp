      Vehicle Fault Dialog

      Record and edit vehicle faults. The fault level assesses the fault service level and has a direct effect on dispatching.

      Overview

      The Vehicle Fault dialog allows authorized users to create and edit vehicle faults in PEAK.DMS. Each fault is assigned a fault level, which reflects the operational fault service level and is considered in dispatching and views.

      Faults can be categorized using a vehicle fault tree (a configured fault catalog). The catalog is stored specific to the vehicle type and can be customized by the administrator.

      Opening the Dialog

      To open the Vehicle Fault dialog:

      Create a New Fault

      • Left-click on the Vehicle Fault button in the "Dialog" toolbar
      • View menu → corresponding entry
      • Vehicle Faults view: Toolbar → Add button
      • Vehicle Faults view: Context menu → Add…

      Edit an Existing Fault

      • Vehicle Faults view: Double-click on a row
      • Vehicle Faults view: Context menu → Edit…

      Note: If the dialog is opened via the context menu of a vehicle, the vehicle number is pre-filled.

      Dialog Structure

      The dialog is divided into two tabs:

      • Basic Data: Vehicle identification, fault categorization, fault level, message data
      • Details: Detailed descriptions, action notes, remarks

      Fields and Meaning

      "Basic Data" Tab

      Vehicle Identification

      FieldDescription
      VehicleNumber of the selected vehicle. Pre-filled if opened from vehicle context.
      TypeVehicle type for the vehicle (display only).

      Category – Recording the Vehicle Fault

      FieldDescription
      Vehicle fault tree

      Selection of a fault category in the predefined vehicle fault tree. Fault types are provided with uniform numbers (quick entry possible).

      Stored in the fault category:

      • Operational fault service level of the vehicle
      • Missing capability
      • Action note

      After selection, the "Fault level" and "Fault text" fields are filled according to the configuration.

      Negated capabilityMissing capability when the selected fault occurs (display only).
      Dispatch vehicle typeCalculated/suggested alternative replacement vehicle type after entering the fault. Can be adjusted manually.

      Tip: Navigate the vehicle fault tree as precisely as possible to achieve accurate categorization. Alternatively, you can enter the fault number directly.

      Fault – Properties

      FieldDescription
      Fault levelOperational fault service level of the vehicle. See Fault Levels.
      Message numberNumber for unique identification of the vehicle fault (optional, textual identifier, freely definable).
      StateCurrent processing status: New, Reported, In Progress, Completed, No Fault Found. See Fault States.
      CreatorUser who recorded the fault (display only).
      Reported atTime of the fault report (display only).
      Fault textDetailed description of the defect/damage (free text). Is pre-filled from the vehicle fault tree, can be adjusted.
      Action noteNotes for garage/service (free text). Is pre-filled from the vehicle fault tree.
      RemarksOptional comment (free text).

      "More" Button

      If a document with detailed action notes is stored for the fault category, it can be displayed by pressing the More button. The button is disabled if no document is stored.

      Options – Handling the Vehicle Fault

      OptionDescription
      Create garage orderCheckbox: Create a garage order to resolve this fault.
      Add all open faultsCheckbox (active when "Create garage order" is checked): All reported faults for the vehicle are assigned to the garage order to be created.
      Create vehicle reservationCheckbox: Convert fault directly into a vehicle reservation after recording.
      Include all faults from vehicle Checkbox (active when "Create vehicle reservation" is checked): Include all open faults in the vehicle reservation.
      CompletedCheckbox: Transition the vehicle fault to the "Completed" state. Only active in "Reported" or "In Progress" state.
      No Fault FoundCheckbox: Transition the vehicle fault to the "No Fault Found" state (after unsuccessful inspection). Only active in "Reported" state.

      Note: The "Completed" and "No Fault Found" checkboxes are inactive when first recording a fault.

      "Details" Tab

      FieldDescription
      DescriptionDetailed description of the damage/defect.
      Short text (Driver info)Max. 40 characters short info for driver terminals. Is displayed in: Vehicle Dossier (next to vehicle state), Vehicle List, Driver Departure Board (block information). Field is linked to user permission.
      Action noteNotes for garage/service.
      RemarksOptional comment.

      Best Practice: Formulate the short text concisely and meaningfully, as only 40 characters are available.

      Workflows

      Create a New Fault

      To create a new vehicle fault:

      Prerequisite: "Vehicle Fault" dialog is open in "New" state.

      1. Select Vehicle (may already be pre-filled if called from context).
      2. Vehicle fault tree: Select fault category
        • Via tree navigation by opening branches or
        • Via quick entry of the fault number

        The system automatically fills in: Fault level, Fault text, Action note, Negated capability, Dispatch vehicle type.

      3. Check Fault level and adjust manually if necessary.
      4. Enter Message number (optional).
      5. Check and supplement Fault text.
      6. Enter Short text (SSB, optional, max. 40 characters).
      7. Check and supplement Action note (optional).
      8. Enter Remarks (optional).
      9. Optional: Activate Create garage order
        • If needed: Activate Add all open faults
      10. Optional: Activate Create vehicle reservation (SSB).
      11. Left-click on OK.

      Result:

      • Fault is saved in "Reported" state.
      • Dialog is closed.
      • Fault appears in "Vehicle Faults" view and related views.
      • Vehicle may no longer be available for operation according to the fault level.
      • If the option is activated, the "Garage Order" or "Vehicle Reservation" dialog opens.

      For detailed information, see Create Vehicle Fault.

      Edit Fault

      Editing a fault allows properties to be changed subsequently. The dialog fields available for editing depend on the current state of the fault.

      Fault in "Reported" state:

      • All fields editable as in new creation
      • "Completed" and "No Fault Found" checkboxes are active
      • Garage order can be created

      Fault in "In Progress" state:

      • Editable: "End", "Organizational unit"
      • Many fields are locked for subsequent changes
      • "Completed" checkbox is active

      For detailed information, see Edit Vehicle Fault.

      Create Fault-Related Garage Order

      A garage order can be created for one or more faults of a vehicle. The fault changes from "Reported" to "In Progress".

      For detailed information, see Create Garage Order for Fault.

      Report Fault as Completed

      A fault can be transitioned to the "Completed" state manually or automatically.

      Manual ways:

      • "Vehicle Faults" view: Context menu → Report finished / Completed
      • "Vehicle Fault" dialog: Activate Completed checkbox

      Automatic: Fault is automatically set to "Completed" when the associated garage order changes to the "Finished" state.

      For detailed information, see Report Vehicle Fault as Completed.

      Cancel Fault (No Fault Found)

      A fault can be transitioned to the "No Fault Found" state after an unsuccessful inspection. This action requires special permission.

      Ways:

      • "Vehicle Faults" view: Context menu → Cancel
      • "Vehicle Fault" dialog: Activate No Fault Found checkbox

      System Effect

      Effect on Dispatching

      • The fault level controls the availability of the vehicle in automatic block dispatching.
      • Manual dispatching takes fault levels into account.
      • Vehicles can be fully or partially blocked depending on the fault level.

      Aggregation with Multiple Faults

      • Views display the highest (most critical) fault level.
      • Dispatching also uses the highest fault level of all faults on a vehicle.

      Display in Views

      Fault data appears in:

      • Vehicle Faults View: All faults with status and fault level
      • Vehicle Dossier: Faults for the vehicle, short text next to vehicle state
      • Vehicle List: Fault level display, short text
      • Driver Departure Board (SSB): Short text in block information
      • Fault-related lists: Depending on configuration

      Links

      • Garage Orders: Fault can trigger a garage order; state change on completion
      • Vehicle Reservations (SSB): Fault can trigger a vehicle reservation

      Best Practices

      • Vehicle fault tree: Navigate as precisely as possible for accurate categorization
      • Fault text: Text from catalog can and should be manually supplemented if necessary
      • Fault level: Careful selection, as it has a direct effect on dispatching; if uncertain, choose a higher (more critical) fault level
      • Multiple faults: If a vehicle has multiple faults, consider "Add all open faults" (avoids multiple parallel garage orders)
      • Documentation: Record action notes completely; use remarks for communication between shifts/departments
