      The "Garage Orders" View

      The "Garage Orders" tabular view shows all planned, ongoing, and completed garage orders. Each order is assigned to exactly one vehicle .

      Opening the View

      You can open the list of garage orders (vehicle reservations) in the following ways:

      • Via the "Data Views" toolbar by clicking the Garage Orders button.
      • Via the menu: View > Data Views > Garage Orders .

      Open via menu/toolbar or by double-clicking from the "Garage Orders" view or via the context menu of a vehicle. The associated "Garage Order" dialog is used for time and spatial planning (vehicle, order type, time period, priority, location/organizational unit, assign faults).

      Columns (Default Configuration)

      • Vehicle: Vehicle number of the reserved vehicle.
      • Vehicle Type: Vehicle type of the reserved vehicle.
      • Start: Planned start of the garage reservation (operating day and time).
      • End: Planned end of the garage reservation (operating day and time).
      • State: Current processing state of the order (e.g., Planned, In Progress, Finished, Canceled).
      • Depot: Depot where the work is taking place.
      • Order Type: The type of garage order.
      • Priority: Priority of the order execution (1-10, where 10 is the highest).
      • Last modified by: User who last edited the order.
      • Last modified at: Time of the last edit.

      Context Menu & Actions

      • Add – Create garage order (opens dialog)
      • Edit – Change order properties
      • Finish – Finish order (State "Finished")
      • Cancel – Cancel order (State "Canceled")

      Other actions (e.g., in the dialog): Add/Remove Fault , state change depending on the current state (e.g., more fields editable in "Planned" state; "End"/"Organizational unit" editable in "In Progress" state).

      States & Dispatching Effect

      States indicate the processing status and can be set manually or via an interface: Suggested (without effect on dispatching), Planned , In Progress , Finished , Canceled . During the reservation period, an order blocks the vehicle for dispatching (depending on order type/state). "Finished"/"Canceled" make the vehicle serviceable again; with "Finished", assigned faults are completed, with "Canceled", they revert to "Reported".

      Dispatching only considers orders in the Planned state (if the period is not exceeded) or In Progress ; priority 1–10 (Must/Can reservation) controls the effectiveness.

      Integration in Other Views

      • Vehicle Dossier – "Garage Orders" tab shows past, current, and planned orders related to the vehicle.
      • Vehicles – Context menu entries lead directly to creating/opening garage orders.

      Brief Overview: "Garage Order" Dialog

      Typical fields: Vehicle, Order Type, Type of work (free text), Start/End, Priority, State, Remark, Location of work (Depot/Parking space/Organizational unit) ; integrated tables for "Faults to be resolved" and "Existing garage orders".
