      Process integration (positioning & telemetry)

      Process integration groups real-time interfaces through which PEAK.DMS continuously receives data from operations and can act on operations — such as positioning, states, and signals.

      Data from operations (inbound)

      • Positioning on route — periodic transmission of current vehicle positions/orientations via online interface.
      • Positioning on the depot — position determination via radio signals; handover to PEAK.DMS.
      • Vehicle/train status — states according to process data; part of the operational data model.
      • Positioning points/RFID/transponders for entry/exit and depot areas
      • Vehicle telemetry — status, energy, diagnostics with plausibility checks.
      • Events from peripherals e.g., charging points, VSP terminals.

      The data flows into dispatching, lineup and stabling logic (e.g., automatic stabling allocation, e‑bus charging windows, priorities per stabling position).

      Impact on operations (outbound)

      Via process integrations, PEAK.DMS can influence processes in dispatching terms (e.g., displays/signals).

      • Depot/stop displays — driver/traveler information based on the current operating situation.
      • Transmission of target stabling positions/route releases to drivers/terminals.
      • Dispatching decisions (rotation, vehicle swap) to Control Center systems.
      • Releases to charging/supply infrastructure.

      Relation to the data model

      All process data is stored consistently in the central operational data model (timetable status, forecasts, location, device states). This makes it uniformly available in views/dialogs.
