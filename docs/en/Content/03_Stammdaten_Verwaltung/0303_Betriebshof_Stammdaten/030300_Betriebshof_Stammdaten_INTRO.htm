      Depot master data

      Master data for the depot includes depot topology (parking positions, tracks), states and usage types, and overarching parameters per depot. They form the basis for dispatching, visualization, and reports in DMS.

      Definition & overview

      • “Depot” module: Information on operations and current vehicle inventory of a depot.
      • Vehicle assignment: Vehicles are assigned to a home depot or loaned depot.
      • Administration: Tabular views and dialogs for creating and maintaining master‑relevant properties.

      “Depots” view

      Tabular overview of all depots configured in PEAK.DMS. Typical columns:

      • Depot (designation), Abbrev.
      • Vehicle inventory (Planned), Vehicle inventory (Actual)

      “Depot information” dialog

      Maintain central depot parameters:

      • Driver information (text for driver terminal)
      • Vehicle inventory (planned/actual) (info fields)
      • Diesel/Heating‑oil refueling optimization (checkboxes)
      • Responsibility (host name for event output)

      Parking positions

      Parking positions are topological elements of the depot.

      Types of parking positions

      • Single parking position
      • Segment of a track
      • Staging area

      Parking position states

      • Available — considered by automatic dispatching
      • Blocked — ignored by automatic dispatching

      Parking usage types & day types

      The parking usage type defines the operational purpose (e.g., block, workshop, supply, reserve) and links parking properties with vehicle/vehicle type.

      • Configure per parking position for a period and with a priority.
      • Multiple usage types can be grouped into a day type.

      Special areas in the depot

      • Staging “outsourced” (bus): staging area for vehicles in an external workshop; vehicles parked there are excluded from positioning.
      • Pull‑in monitor (rail): shows next pull‑ins at the depot entrance (approx. 15 min before predicted/planned arrival).
      • Invalid locations: area for vehicles with implausible location (e.g., multiple occupancy, missing exit detection).

      Relation to dispatching

      • Automatic vehicle dispatching considers usage types, parking states, and depot‑specific rules.
      • Vehicle type and state criteria (e.g., incidents, supply/workshop) control the parking position assignment.
