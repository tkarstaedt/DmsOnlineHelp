      Overview of dialogs

      Dialogs in PEAK.DMS are used to enter, edit, and confirm data. Unlike views, which primarily display information, dialogs request your active input and confirm changes before saving.

      Dialog types

      • Modal: Locks the UI until you finish (OK, Cancel).
      • Modeless: Remains open while you keep working in views.
      • Wizards (multi‑step): Guide you step by step through complex input.
      • Confirmation and information dialogs: Ask for confirmation or report results.

      Open dialogs

      • Open dialogs from views via double‑click on an object or via the context menu.
      • Use buttons in the view toolbar or entries in the menu bar.

      Interpret buttons

      • OK: Validate and apply input; close the dialog.
      • Apply: Apply input without closing the dialog.
      • Cancel/Close: Close the dialog and discard changes.
      • Help: Open context‑sensitive help for the open dialog.
