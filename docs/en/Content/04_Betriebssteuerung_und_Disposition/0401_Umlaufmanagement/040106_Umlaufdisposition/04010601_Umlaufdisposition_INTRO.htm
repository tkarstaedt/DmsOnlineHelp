      Block Dispatching  –  Introduction

      The block dispatching assigns planned blocks to suitable vehicles and makes the result visible in the stabling lineup plan and the depot view.

      Purpose and Differentiation

      The block dispatching assigns blocks to suitable vehicles based on the current depot situation. The task is to assign a vehicle to every planned block and to ensure smooth operations control.

      It must be differentiated from vehicle/stabling position dispatching (physical assignment Vehicle↔Stabling Position) and represents the logical assignmentBlock↔Vehicle, indirectly influenced by the stabling position location.

      Considered Factors

      The block dispatching considers the following factors for the assignment:

      • Vehicle Properties: Vehicle type, range/SoC, disturbance level, line restrictions
      • Planned Reservations of the vehicle to be dispatched
      • Current Stabling Position and topological dependencies on the depot
      • Stabling Position Properties: Track lengths, charging points, authorized usage, pull-out logic
      • Block Properties: Vehicle type, depot (start/end), pull-in/pull-out time
      • Configured Dispatching Conditions: Restrictions, criteria, priorities

      Operating Modes

      • Manual: The dispatcher assigns a vehicle to the block to be dispatched and confirms changes.
      • Semi-automatic: The user initiates the block dispatching in the "Block Dispatching" dialog; the system calculates suggestions automatically; confirmation by dispatching.
      • Fully automatic: Occurs at regularly recurring, configurable intervals (e.g., every 3–5 minutes); the system independently implements released rules; suggestions can be checked/overridden.

      Dispatching Process

      The block dispatching calculates the vehicle assignments in sequential order, beginning with the block starting at the earliest time. Blocks starting immediately are assigned to operationally ready vehicles first.

      Planning Period

      The planning period begins at the time of dispatching execution and ends at the end of the current pull-out phase.

      Re-dispatching and Block Fixation

      In case of a changed depot status (vehicle positions, incidents, reservations, special trips), the block dispatching can re-dispatch blocks already assigned to other vehicles.

      Block Fixation: Block assignments are not broken if:

      • the block is within the configurable block fixation time (e.g., 25 minutes before block start) or
      • the block assignment was manually made by a user-scheduled vehicle pre-reservation.

      Control Core

      The dispatching is executed per depot, operating day, and pull-out phase. Only depots in *Free Stabling* mode are selectable.

      • Start Dispatching: Calculation based on the current depot occupancy; result check in the stabling lineup plan and the depot view.
      • Reset Blocks: cancels generated assignments; blocks in the *In Progress* status remain assigned.
      • Change Pull-out Phase: Dispatching is recalculated with current occupancy.
      • Permissions: observe required rights to start/delete block dispatching.

      Vehicle Shortage

      If the block dispatching calculates a Vehicle Shortage, the first unassigned block starts at the latest possible time. This grants operations the greatest possible scope for action.

      • In case of vehicle shortage: blocks starting at the latest time are not assigned.
      • The authorized user is notified of the vehicle shortage via a message.
      • The preview window for vehicle shortage is configurable.

      Cancellation of Block Assignments

      Automatically Calculated Assignment

      An automatically calculated block assignment is canceled if the planned block begins outside the configurable block fixation time and one of the following cases occurs:

      • the dispatched vehicle leaves its current stabling position,
      • the dispatched vehicle is blocked by a disturbed vehicle in front of it (pulling out not possible),
      • the dispatched vehicle is reported as disturbed.

      Note: The assignment remains if the vehicle is reported as disturbed and the block starts within a configurable period (e.g., ≤ 15 min).

      Manually Set Assignment

      A manually set block assignment can only be canceled by the authorized user.

      Cancellation and Maximum Duration

      The Maximum Duration, for which the block dispatching is executed automatically, is configurable. If a running block dispatching reaches the configurable maximum duration, it is canceled.

      • The result of the calculated assignments may be incomplete.
      • An incomplete result has the consequence that not all planned blocks could be assigned a vehicle.

      Result Presentation

      • Stabling Lineup Plan: Assignment of blocks to tracks/stabling positions with the associated vehicles; sequence reflects pull-out logic.
      • Depot View: Vehicle symbols show successful block assignments color-coded; visual stabling position occupancy with status and warning notes (e.g., conflicts, workshop).
      • Tabular Views: show the current block assignment to a vehicle.

      Interaction with Duty Lineup Planning

      The Duty Lineup Planning forms the planning basis (scenarios), the operative duty lineup plan the day-specific implementation, and the stabling lineup plan the graphical control incl. consistency checks. Rules such as type admissibility and usage types flow into dispatching.

      Best Practices

      • Check preview periods in dispatching views and area calendars before the start of the shift; document manual interventions.
      • Maintain rules/priorities of the automatic system and check suggestions with scoring; only override when targeted.
      • Coordinate conflicts with workshop/supply and pull-out logic in the stabling lineup plan; use consistency checks.
      • Adjust block fixation times and preview windows to operational requirements.
      • React early in case of vehicle shortage; observe automatic messages.

      SSB – Project-Specific Notes

      • Operating modes and priorities/rules (line/block priorities, vehicle classes, deployment window) are project-specifically parameterized; confirmation levels/information chains follow internal procedures.
      • Restrictions from the duty lineup plan (e.g., tunnel operation, vehicle lengths, energy/charging window) control the automatic assignment and later the rotation.
      • Block fixation time, maximum duration, and preview window are customer-specifically configured.
