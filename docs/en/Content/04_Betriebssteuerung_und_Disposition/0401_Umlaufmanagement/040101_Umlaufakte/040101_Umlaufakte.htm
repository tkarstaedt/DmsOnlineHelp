      Block dossier dialog

      Use the block dossier dialog to centrally display and edit all information about a block.

      Open the dialog

      • Click the Block dossier toolbar button.
      • Double-click the block in the tabular Blocks view.

      The dialog fields are grouped into tabs. The tabs are arranged horizontally and can be selected with a click.

      Tab Description
      Basic data Shows the properties of the selected block and allows editing them.
      Block segments Displays the segments of the selected block.
      Vehicle types Display and assignment of vehicle types permitted for the selected block.
      Messages Shows the messages for the selected block.

      Table 65: Block dossier dialog — overview of all tabs

      Context fields above the tabs

      • Date — operating day on which the block is performed.
      • Block — number of the block.

      Tab “Basic data”

      The Basic data tab shows the properties of the selected block.

      Figure: Block dossier — Basic data tab (example)

      Field Description
      Block type Block type related to the vehicle.
      State Current state of the block.
      Start Code of the depot where the first duty of the block is planned to begin.
      Start time Time when the vehicle is planned to pull out and the first duty of the block begins.
      Target Code of the depot where the last duty of the block is planned to end.
      End time Time when the vehicle is planned to pull in and the last duty of the block ends.
      Length Length of the block in kilometers.
      Current vehicle

      Current vehicle

      The button opens the Current vehicle dialog.
      Planned vehicle

      Planned vehicle

      This information is used for logging in PSItraffic.
      User-scheduled vehicle

      Vehicle assigned to the future block by an authorized user.

      The user-scheduled vehicle is an input to automated block dispatching; the block is to be assigned with this vehicle.
      Execution note

      Free text.

      The entered text informs the driver and is shown on the departure board.

      Table: Block dossier — Basic data tab

      Tabs Current vehicle and Planned vehicle

      Current vehicle

      Information about the vehicle configured to carry out the block.

      • Block — number of the block for which the vehicle is configured.
      • Vehicle — identifier of the vehicle used to perform the current block.
      • Assignment type — process used to assign the vehicle to the block.
      • Valid from — point in time when the assignment becomes valid.
      • Cancellation — point in time when the assignment ceases to be valid.

      Planned vehicle

      The planned vehicle is set during pre-planning and can be used during the operational phase.

      • Block — number of the block for which the vehicle is configured.
      • Vehicle — identifier of the planned vehicle intended for this block.
      • Valid from — point in time when the assignment becomes valid.
      • Cancellation — point in time when the assignment ceases to be valid.

      Tab “Block segments”

      The integrated view shows past, current, and planned block segments of the selected block according to the configured data retention in the operational system.

      Figure: Block dossier — Block segments tab (example)

      Field Description
      Vehicle Current vehicle
      Start point Depot / stop where the segment begins.
      End point Depot / stop where the segment ends.
      Start (planned) Planned start time at the start point.
      Start (actual) Actual start time at the start point.
      End (planned) Planned end time at the end point.
      End (actual) Actual end time at the end point.

      Table: Block dossier — Block segments tab (default configuration)

      Tab “Vehicle types”

      The Vehicle types tab lists the vehicle types for the selected block that automated block dispatching considers according to their priority.

      Figure: Block dossier — Vehicle types tab (example)

      • Assign permitted vehicle types including priority as input for block dispatching.
      • Alternatively select and apply a vehicle type pattern; then adjust types/preferences as needed.

      If you reapply a vehicle type pattern, previously edited entries in the list are overwritten.

      Tab “Messages”

      The integrated view shows past and current messages related to the selected block according to the configured data retention.

      Figure: Block dossier — Messages tab (example)

      Field Description
      Timestamp Date and time of creation.
      Type Message type.
      State Current processing status of the message.
      Depot Depot to which the message refers.
      Vehicle Number of the vehicle referenced by the message.
      Created by User who created the message.

      Table: Block dossier — Messages tab (default configuration)

      State-dependent editing

      • Planned: Full editing of all relevant fields possible.
      • In progress: Only Target, End time, and Current vehicle are editable.
      • Finished: Read-only; no further editing.

      Typical actions

      • Assign/swap vehicle: Set the current vehicle; use the advanced swap dialog (time-based distinction between swap/change).
      • Maintain vehicle types: Add/remove types with priority or apply a pattern.
      • Cancel/reactivate block: Change status; reactivation generates a message.
      • Alternative assignment via the stabling lineup plan: use Set location to place the vehicle at the intended parking position; the system assigns the block.

      SSB — project-specific note

      The basis of automatic dispatching is the stabling lineup plan (duty tiles), not the block data shown in the Blocks view.
