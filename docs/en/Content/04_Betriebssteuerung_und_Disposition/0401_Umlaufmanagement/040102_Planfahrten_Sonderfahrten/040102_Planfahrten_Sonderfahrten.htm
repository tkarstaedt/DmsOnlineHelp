      Special block order — Planned trips and special trips

      Planned trips come from timetabling; you create special trips as special blocks directly in DMS (exactly one block segment). Capture via Special block order, with series options and clear integration into operational views.

      Overview

      Planned trips are imported from timetabling via the interface and dispatched in block management. Special trips (special blocks) are created directly in PEAK.DMS as needed and consist of exactly one block segment.

      Create special trips

      Open the Special block order via the toolbar or the block list. The dialog is structured into the tabs Basic data, Vehicle requirements, and Vehicle supplies.

      Open the dialog: Toolbar → Special block order button (alternatively via the block list).

      Key fields

      Field Description
      Start/End Calendar date and time of the special trip; trip duration results from start/end.
      Block number Identifier from a dedicated number range; default suggestion is prefilled.
      Block usage type Track usage type for dispatching; links the block and parking position.
      Start/End (depot) Select start and end depot.
      Trip destination Destination of the first duty; shown on driver terminal/departure board if configured.
      Pull-out direction Direction in which the vehicle pulls out; can be a criterion for automatic block dispatching (configuration-dependent).
      Execution note Free text; information for the driver and, if applicable, the departure board.
      Additional note Free text; displayed in the Blocks view if configured.
      Reservation reason Purpose of the block.
      Length Block length in kilometers.
      Dwell times Cumulative dwell times (e.g., BEV) in hh:mm format.
      Vehicle reservation — inputs to automated block dispatching
      Vehicle User-scheduled vehicle.
      Vehicle type Permitted vehicle type.
      Vehicle type pattern Permitted vehicle type pattern.
      Vehicle must be full If checked: only fully fueled/charged vehicles are permitted (restriction for automation). If unchecked: no restriction regarding level of fuel/charge.
      Current vehicle Reference to the currently assigned vehicle (context link in the application).
      Tab Basic Data

      Allows the input and editing of the properties of the selected **special block** (**Sonderumlauf**) as well as **vehicle-related specifications** (**fahrzeugbezogener Vorgaben**).

      Field Description
      Trip Properties  –  Properties of the **special block**
      Start Calendar day and time when the **special block** starts.
      End Calendar day and time when the **special block** ends.
      Generate daily until Unchecked: one-time event. Checked: repeating event; daily generation until the specified date.
      Block Identifier as a numeric sequence of digits; **number range** (**Nummernkreis**) for **special blocks** is separate from **planned blocks** (**Planumläufen**).
      Block Usage Type Track **usage type** (**Spur‑Nutzungsart**); restriction for **automated block dispatching** (**automatisierte Umlaufdisposition**).
      Start Abbreviation of the **start point** (**Startpunkts**) where the **special block** is planned to begin.
      Target Abbreviation of the **end point** (**Endpunkts**) where the **special block** is planned to end.
      Destination Destination of the first trip; informational display on the bus display, if implemented via interface.
      Pull-out Direction Direction in which the **vehicle** pulls out; can be a criterion for **automated block dispatching** (**automatisierte Umlaufdisposition**) (configuration-dependent).
      Execution Note Free text; information for **driver** and possibly **departure board** (**Abfahrtstafel**).
      Additional Remark Free text; is displayed in the *Blocks* (**Umläufe**) view, if configured.
      Reservation Reason Purpose of the **block**.
      Length **Block length** in kilometers.
      Layover Times Accumulated **layover times** (e.g., e-vehicle) in hh:mm format.
      Vehicle Reservation  –  Specifications for **automated block dispatching**
      Vehicle **Vehicle** intended by the **user**.
      Vehicle Type **Permissible vehicle type**.
      Vehicle Type Pattern Permissible **vehicle type pattern**.
      Vehicle must be fully refueled Checked: only **fully refueled vehicles** are permissible (**restriction** for the **automatic system** (**Automatik**)). Unchecked: no restriction regarding **fuel level** (**Füllstand**).
      Current Vehicle Reference to the **currently assigned vehicle** (**aktuell zugeordnete Fahrzeug**) (**context link** in the application).
      Tab Vehicle requirements

      Select capabilities that the vehicle to be used must fulfill. These can be criteria or restrictions for automated block dispatching.

      Tab Vehicle supplies

      Information about vehicle-related supplies:

      • After the special block ends and the vehicle enters the depot — inputs to automated vehicle dispatching.
      • Before the special block begins — informal inputs.
      Tab Messages

      Enter textual messages and notes related to the special block.

      Relation to duty lineup plan and stabling lineup plan

      • Not in duty lineup plan: special trips are not listed in the duty lineup plan.
      • Assign operationally: parking occupancy is done automatically by dispatching or manually in the operational duty lineup plan/stabling lineup plan by assigning from the block list.

      Block usage type and track usage type

      The block usage type links block and parking position. Special trips (or special usage types such as staff vehicle) may only start from tracks whose usage type has matching validity.

      States (overview)

      • Block: Planned, In progress, Finished, Cancelled.
      • Trip (ITCS/RBL): Registered, Block is registered, Deregistered, Open, On demand.

      SSB — project-specific notes

      • Pull-out direction supports choosing the exit gate and searching a parking position; special trips are created by calendar date, the operating day is derived automatically.
      • Optional inputs such as Vehicle must be full and other parameters are project-specific.
      • Preview/planning horizon and editing restrictions (e.g., start date) depend on configuration.
