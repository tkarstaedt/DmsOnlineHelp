      View “Pull-in list” - Depot arrivals

      The Pull-in list is the central tabular view for planned, ongoing, and completed pull-ins per operating day and depot — with filters, period controls, and drill-down to dispatching dialogs.

      Overview

      The Pull-in list is a variant of the Block list with focus on pull-ins. It supports control of incoming block segments and preparation of stabling/continued use in the depot. A block segment may end at the depot, but it does not have to.

      Access and filters

      • Menu: View → Data views → Pull-in list
      • Toolbar: Pull-in list button
      • Filters: Operating day, Depot
      • Period: via No. of days show additional following days; optional automatic advance on operating-day change.

      Typical columns (standard configuration)

      Column Description
      Vehicle Current vehicle for the block segment.
      Block Block number.
      Depot Code of the depot where the segment is planned to end.
      Pull-in time (planned) Planned entry time (operating day/time).
      State Current state of the block segment; matches the state of the related block.
      Pull-in time (actual) Actual entry time; set by positioning at the depot entrance.

      Operations

      Edit block (drill-down)

      1. Open the context menu of a row with status Planned.
      2. Select Edit → the Block dossier dialog opens.

      Assign parking (dispatch pull-in)

      • Double-click a row with status In progress to open Vehicle dispatching.
      • Alternatively: context menu → Parking dispatching.

      Notes

      • Turn-back dispatching: Vehicles/segments that pull in significantly before the planned block end are treated as turn-backs. Such vehicles are dispatched to tracks with usage type Turn-back; alternatively end the block manually and dispatch normally.
      • The Pull-in list focuses on entries; adjacent dispatching is done in Vehicle dispatching and the stabling lineup plan.
