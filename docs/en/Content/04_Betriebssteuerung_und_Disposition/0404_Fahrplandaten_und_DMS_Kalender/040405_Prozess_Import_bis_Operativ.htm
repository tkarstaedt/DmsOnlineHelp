      Process: Import to Operational

      This process describes the path of timetable data from planning (e.g., via VDV-452) through the base version and release to the operational data foundation of PEAK.DMS.

      Objective

      You can import, check, release, and securely transfer timetable data into operational use – including documentation and rollback capability.

      Roles & Responsibilities

      RoleTask in the Process
      PlanningCreates/delivers the export (e.g., VDV-452); defines day types.
      DispatcherChecks RBL calendar/base versions, assesses impact on dispatching.
      DMS Specialist AdminStarts import, performs quality checks, releases or discards the version.

      Process (Quick Overview)

      1. Import of deliveries from planning (e.g., VDV-452).
      2. Creation of a base version and content review.
      3. Release of the desired version.
      4. Transfer to Operational and check of area views.
      5. Logging and, if necessary, Rollback.

      Steps in Detail

      1. Import

      1. Open the import function and select the delivery package (e.g., VDV-452 data).
      2. Start the import and check the results log for errors/notes.
      3. A base version of the timetable data is created (see Fahrplandaten anzeigen).

      2. Content Review

      • In the RBL calendar and the base versions, check the validity periods and completeness.
      • Check the operating day ↔ day type assignment in the VDV‑Calendar and the DMS Day Types.
      • Assess impacts on the "Duty Lineup Plan" Dialog and Vehicle Dispatching – Introduction.

      3. Release

      1. Select the checked base version.
      2. Grant the release for operational use.
      3. Document the date/time, responsible role, and a comment.

      4. Transfer to Operational

      1. Initiate the transfer to the operational data foundation.
      2. Check the area views (e.g., Calendar: Blocks, Workshop, Supply) for plausibility.
      3. Perform spot checks on the duty lineup plan and dispatching views.

      5. Logging & Rollback

      • Every release/transfer is logged (version, time, role, comment).
      • If necessary, you can revert to the last released version (Rollback).
      • Summarize the reasons and consequences of the reversal in a comment.

      Quality Assurance (Checklist)

      • Complete operating days in the DMS calendar? → see Dialog "Maintain DMS Calendar"
      • Correct day type assignment? → see VDV‑Calendar
      • Area calendars plausible? → Block/Workshop/Supply checked
      • No open errors in the import log?

      Common Problems & Solutions

      • Missing operating days: Update calendar and check again (see Dialog "Maintain DMS Calendar").
      • Incorrect day type: Correct assignment in the VDV calendar.
      • Unexpected operational version active: Check releases; perform rollback if necessary.
