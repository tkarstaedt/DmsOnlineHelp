      VDV‑Calendar

      The VDV‑calendar assigns every operating day to a day type (e.g., Weekday, Saturday, Holiday) for PEAK.DMS. The assignment affects the duty lineup, vehicle dispatching, and area calendars.

      Objective

      You understand the meaning of the VDV‑calendar and can view, check, and, if necessary, adjust the operating day ↔ day type assignment.

      Context

      Day types are defined in planning and transferred via interfaces (e.g., VDV-452). In the DMS, the VDV‑calendar ensures that operational functions use the correct day logic (e.g., vacation timetables, special services on holidays).

      • Operating Day: Date, possibly with an operational day change.
      • Day Type: Classification of the operating day (regular/special days).
      • VDV‑Calendar: Mapping Operating Day ↔ Day Type.

      Menu paths/designations may vary depending on the project. This topic describes the generic product scope.

      Prerequisites

      • You are signed in to the DMS client and have viewing permissions for timetable data.
      • The system contains day types and, if applicable, imported calendar data.

      View VDV‑Calendar

      1. Open the Timetable Data function.
      2. Select the VDV‑Calendar view.
      3. Filter by Period or Day Type to narrow down the relevant operating days.
      4. Check the columns Date, Day Type, Source, and Status.

      Use column sorting/text filters to quickly identify upcoming special days (e.g., holidays).

      Adjust Assignment

      In exceptional cases, it may be necessary to correct the assignment manually (e.g., short-notice public holidays):

      1. Select the operating day.
      2. Select Change Day Type and set the correct day type.
      3. Save the change and document the reason (comment).

      Manual deviations should be used sparingly and checked after a new import/reconciliation.

      Check & Validate

      • Compare samples with the Fahrplandaten anzeigen.
      • Reconcile the assignment with DMS Day Types.
      • Check a preview period (e.g., 4 weeks) before timetable changes.

      Notes & Best Practice

      • Maintain special days (fairs, events) in planning early so that the correct day type is imported.
      • Document manual changes with date, role, and reason.
      • Synchronize regularly: Planning → Import → Operational check.

      Typical Error Patterns

      • Incorrect day type for a date: Check import status; if necessary, correct manually and update the planning data.
      • Missing assignment: Create/activate the day type in the DMS and perform the import again.
      • Overlaps/Multiple assignments: Clean up duplicates and clarify rules for special days.
