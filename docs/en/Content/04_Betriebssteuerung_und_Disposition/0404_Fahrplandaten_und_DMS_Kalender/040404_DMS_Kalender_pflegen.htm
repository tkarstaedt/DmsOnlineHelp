      Dialog "Maintain DMS Calendar"

      Maintain the DMS calendar: Assign day types to operating days, update the calendar, and account for special days – the basis for dispatching and duty lineup planning.

      Objective and Context

      The DMS calendar maps all operating days and links them with day types. It is the operational basis for dispatching and duty lineup planning. The calendar can be updated regularly and manually adjusted if necessary (e.g., special days, holidays, events).

      The DMS calendar is maintained independently of the VDV-452 calendar but can be based on imported calendar data. At least one default day type must be defined for each depot and day of the week. Manual adjustments (e.g., holidays) are not automatically overwritten.

      Prerequisites

      • Write permissions for calendar maintenance.
      • Optional: Imported day types and VDV calendar data as a basis.
      • A default day type is stored for each depot and day of the week.

      Update Calendar (Automatic and Manual)

      1. Open the DMS Calendar dialog.
      2. Click Update and select the period (e.g., next quarter/year).
      3. Confirm with OK – the calendar is populated with the default day types.
      4. Manual adjustments (e.g., holidays, events) can be made at any time and are not overwritten during automatic updates.

      Note: The calendar must always be maintained at least for the configured preview time of the operational data (e.g., 30 days in advance, 3 days in retrospect).

      Assign Day Types

      1. Select the operating day (Date/Depot).
      2. Select Change Day Type and assign the appropriate day type (e.g., Weekday/Saturday/Holiday).
      3. Save the change.

      Note: Day types are maintained separately (see DMS Day Types). Only one day type can be assigned to each operating day.

      Depot-specific: Different depots can use different day types for the same calendar day.

      Account for Special Days

      • Enter short-term changes (holidays, events, construction site timetables) directly in the DMS calendar.
      • Adjust the day type of the affected operating day if necessary.
      • Manually adjusted calendar days are not overwritten during automatic updates.

      Automatic Creation of Operating Days

      • Operating days are automatically created for a configured preview time. Missing day type assignments trigger warning messages in advance.
      • A scheduled service automatically creates missing operating days and assigns them the last used or the default day type as a standard.
      • Each operating day inherits usage types and permitted vehicle types for stabling positions from the assigned day type.
      • A DMS operating day always has a duration of 24 hours. The time of change can be configured per depot (Default: 00:00).

      Operational Plans per Operating Day

      • From approved duty lineup plans of the day type, operational duty lineup plans are created per operating day (incl. transfer of pull-out stabling positions and block parameters).
      • Blocks are reconciled with the timetabling tool (e.g., DIVA); matching blocks are connected and enriched with duty lineup information.
      • Operational duty lineup plans can only be created for operating days that are assigned a day type.

      VDV-452 Import and Data Versions

      • The DMS calendar is independent of the VDV-452 calendar but can be based on imported calendar data.
      • Each export from the planning system contains a company calendar with the assignment of the day type to each operating day. If there are multiple exports, the base version with the highest number applies for a given day.
      • The timetable generation uses the current, valid base version from the planning database for each operating day.
      • Already created future operating days can be overwritten by new VDV planning data imports if the planning data has changed.

      Important Notes

      • The DMS calendar is the prerequisite for dispatching and duty lineup planning in the DMS.
      • Each operating day can only have one day type (1:1 relation).
      • Manual changes to individual days (e.g., holidays) are not overwritten during automatic updates.
      • At least one default day type must be defined for each depot and day of the week.
      • The preview time (e.g., 30 days in advance) is configurable and should be maintained regularly.
      • A warning message is issued in advance if a day type assignment is missing.
