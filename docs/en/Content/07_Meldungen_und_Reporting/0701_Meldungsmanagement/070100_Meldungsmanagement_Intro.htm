      Message Management

      The message management in PEAK.DMS consolidates system messages, short messages, and the operational logbook. It supports the Control Center in quickly understanding the current situation, acknowledging and documenting events, and communicating purposefully with driving staff.

      What you will do here

      • Monitor, filter, sort, review details, and acknowledge system messages.
      • Create, read, and reply to short messages between Control Center and driving staff.
      • Trace operational events in the operational logbook and add manual entries.
      • Execute frequent standard actions safely and consistently (e.g., acknowledgements).

      Components

      The following views and tasks make up the "Message Management" module:

      • Display view "Messages" — Reference for columns, filtering/sorting, acknowledgement, and detail dialog.
      • View and create short messages — Concept and tasks for communication between Control Center and drivers.
      • View operational logbook — Reference to the chronological logbook including search/filter and export.
      • Standard actions in message management — Steps for acknowledging and in‑depth analysis.

      Typical workflows

      1. Assess situation: Open the messages view — filter by priority/category — focus on critical events (see Display view "Messages").
      2. Process & document: Open details — check cause/objects — acknowledge message — verify/add entry in the View operational logbook.
      3. Communicate: Send or reply to a short message to driving staff (request acknowledgement if needed), see View and create short messages.
      4. Standard actions: Execute recurring actions according to the guideline (see Standard actions in message management).

      Roles and permissions

      • Dispatching/Control Center: Full access to messages, acknowledgements, and short messages.
      • Service/garage coordination: Read/acknowledge relevant messages, logbook entries.
      • Administrators: Control categories, priorities, view columns, and notifications.

      Prerequisites

      • User permissions for message views, acknowledgement, and message sending.
      • Project configuration for message categories/priorities and logbook export.
      • Optional connection of driver terminals for short messages (project‑specific).

      Best practices

      • Only acknowledge messages after investigating the cause; add a comment for traceability.
      • Formulate short messages concisely; require acknowledgement for critical topics.
      • Add essential events to the operational logbook promptly (e.g., decisions, approvals).
