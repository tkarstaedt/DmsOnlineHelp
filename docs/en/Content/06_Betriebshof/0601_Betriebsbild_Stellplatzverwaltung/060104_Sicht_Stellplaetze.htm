      “Parking Positions” View

      The tabular “Parking Positions” lists all parking positions of a in PEAK.DMS. It is used for quick searching, filtering, and editing of parking position data, as well as for navigation to associated dialogs.

      Purpose and Benefit

      • Overview of all parking positions with status (visible/blocked) and usage.
      • Quick entry into maintenance tasks (e.g., or allowed ).
      • Basis for evaluations and comparisons with the .

      Opening the View

      1. Open the navigation tree and select Depot > Parking Positions.
      2. Alternatively: Search for “Parking Positions” and open the from the search results.

      Columns (Standard)

      The available columns are project-specific. Typical columns in the standard configuration:

      ColumnBrief Description
      DesignationName of the parking position.
      TypeSingle, Track Segment, Storage Space.
      Area/TrackAssignment within the .
      VisibleDisplay in (Yes/No).
      BlockedConsider for (Yes/No).
      Reason for BlockShort text.
      Valid From/ToTime period for block/usage.
      Usage Types, , , Reserve, Storage Space.
      Allowed Vehicle TypesTypes/Patterns.
      Zone/Charging PointAssignment (if used).
      NoteShort comment.

      Occupancy Display & Priorities

      The shows priorities and, if applicable, reserved usages (e.g., e-bus charging) per parking position. In the case of competing assignments, priorities and conflict are displayed.

      • Display of Type Permissibility (, length restrictions)
      • Marking of reserved parking positions (charging window, )
      • Warning in case of block/missing remaining length/conflict

      Operation

      Use the standard functions of tabular .

      • Search: Enter a character string, hits are highlighted.
      • Filter: Apply simple/advanced filters (e.g., only blocked positions).
      • Sort/Group: Click column header or activate grouping.
      • Show/Hide Columns: Adapt to information needs.
      • Export/Print: Use list for reporting.

      Context Menu (Selection)

      Actions are available via right-click on a data record.

      • Edit… – opens Edit Parking Position.
      • Block/Unblock – sets Blocked including reason for block/validity.
      • Add/Remove Usage Type – maintains the list.
      • Allow/Remove Vehicle Type – maintains permitted types/patterns.
      • Show in Depot View – centers parking position graphically (if available).

      Typical Work Steps

      1. Check Blocked Parking Positions: Set filter Blocked = Yes → check reasons for block and validity.
      2. Update Usage Types: Select record → Edit… → maintain Usage Types tab.
      3. Maintain Allowed Vehicle Types: Select record → Edit… → Vehicle Types tab.

      Configuration Notes

      • Column selection and order can be saved per user or profile.
      • Color logic for is visualized in the (project-specific).
      • For e-buses, optional columns for charging infrastructure (e.g., charging point/connector).
